0. 数据处理：建模前的必修课

为什么数据处理是建模的第一步？真实数据往往是“脏”的：量纲不同、缺失值、异常值、类别变量、高维冗余。Garbage In → Garbage Out（GIGO），不处理数据直接建模，结果会失真。

标准流程：

  1. 探索性数据分析（EDA）
  2. 清洗：缺失值、异常值
  3. 变换：缩放、归一化、对数变换
  4. 编码：类别变量数字化
  5. 降维 / 聚类：PCA、K-Means

1. K-Means 聚类：发现隐藏分组

Block 1：K-Means 基本流程

场景：政府要规划公共医院位置，让居民到最近医院的平均距离最小。K-Means 就是这样一个“找中心再分配”的迭代算法。

算法步骤：

  1. 随机放置 K 个中心（centroids）
  2. 每个点分配到最近的中心
  3. 把中心移动到分配给它的点的均值位置
  4. 重复 2-3 直到收敛

目标函数（WCSS）：

WCSS = Σi Σj∈Ci ||xj - μi||²

启发问题：K-Means 对初始中心敏感吗？如果数据本身是长条形状而非球形，聚类效果会怎样？

import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import make_blobs
from sklearn.cluster import KMeans

# --- Parameters (user-tunable) ---
np.random.seed(42)
n_samples = 300
n_clusters = 3
centers = [[0, 0], [5, 5], [0, 5]]

# --- Generate synthetic 2D blobs ---
X, true_labels = make_blobs(n_samples=n_samples, centers=centers,
                            cluster_std=1.0, random_state=42)

# --- Fit K-Means ---
kmeans = KMeans(n_clusters=n_clusters, random_state=42, n_init=10)
labels = kmeans.fit_predict(X)
centroids = kmeans.cluster_centers_

# --- Visualization ---
fig, axes = plt.subplots(1, 2, figsize=(13, 5))
axes[0].scatter(X[:, 0], X[:, 1], c=true_labels, cmap='viridis', edgecolor='k', s=30)
axes[0].set_title('True Groups')
axes[1].scatter(X[:, 0], X[:, 1], c=labels, cmap='viridis', edgecolor='k', s=30)
axes[1].scatter(centroids[:, 0], centroids[:, 1], c='red', marker='X', s=200, label='Centroids')
axes[1].set_title(f'K-Means Clustering (K={n_clusters})')
axes[1].legend()
plt.tight_layout()
plt.show()

print(f'WCSS (inertia): {kmeans.inertia_:.2f}')

Block 2：肘部法则选 K

如何选择 K？不能拍脑袋。肘部法则（Elbow Method）通过比较不同 K 的 WCSS，找到“再增加聚类收益很小”的拐点。

报告写法：“We chose K = 3 as it was the elbow point, minimizing complexity while maximizing cohesion.”

启发问题：肘部不明显时怎么办？轮廓系数（Silhouette Score）能补充判断吗？

import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import make_blobs
from sklearn.cluster import KMeans

# --- Parameters (user-tunable) ---
np.random.seed(42)
X, _ = make_blobs(n_samples=300, centers=5, cluster_std=1.2, random_state=42)
K_range = range(1, 11)

# --- Compute WCSS for each K ---
wcss = []
for k in K_range:
    kmeans = KMeans(n_clusters=k, random_state=42, n_init=10)
    kmeans.fit(X)
    wcss.append(kmeans.inertia_)

# --- Elbow plot ---
fig, ax = plt.subplots(figsize=(8, 4))
ax.plot(K_range, wcss, marker='o')
ax.set_xlabel('Number of clusters K')
ax.set_ylabel('WCSS (Inertia)')
ax.set_title('Elbow Method for Choosing K')
ax.grid(True, linestyle='--', alpha=0.5)
plt.tight_layout()
plt.show()

Block 4：PCA 主成分与方差解释

问题：真实数据特征很多（GDP、人口、收入、教育等），高维导致可视化困难、计算复杂、噪声冗余。PCA 通过协方差矩阵的特征分解，找到保留最多方差的新坐标轴。

核心概念：

  • 协方差矩阵：描述特征两两之间的变化关系
  • 特征向量：主成分方向（新的坐标轴）
  • 特征值：该主成分解释的方差大小

启发问题：PCA 前是否必须先做标准化？如果不标准化，哪个特征会主导主成分？

import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import load_iris
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA

# --- Parameters (user-tunable) ---
n_components = 2

# --- Load and standardize data ---
iris = load_iris()
X = iris.data
y = iris.target
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# --- Apply PCA ---
pca = PCA(n_components=n_components)
X_pca = pca.fit_transform(X_scaled)

# --- Results ---
print(f'Explained variance ratio: {pca.explained_variance_ratio_}')
print(f'Cumulative explained variance: {np.cumsum(pca.explained_variance_ratio_)}')

# --- Visualizations ---
fig, axes = plt.subplots(1, 2, figsize=(13, 5))
axes[0].scatter(X_pca[:, 0], X_pca[:, 1], c=y, cmap='viridis', edgecolor='k')
axes[0].set_xlabel('PC1')
axes[0].set_ylabel('PC2')
axes[0].set_title('Iris Data Projected onto PC1 & PC2')

axes[1].bar(range(1, len(pca.explained_variance_ratio_) + 1),
            pca.explained_variance_ratio_, color='steelblue', edgecolor='black')
axes[1].set_xlabel('Principal Component')
axes[1].set_ylabel('Variance Explained')
axes[1].set_title('PCA Scree Plot')
plt.tight_layout()
plt.show()

Block 5：载荷与 Biplot

PCA 的两个输出：

  • Loadings（载荷）：每个原始特征对主成分的贡献。绝对值大意味着该特征很重要。
  • Scores（得分）：每个样本在新坐标系下的坐标，可用于绘图或后续建模。

启发问题：如果 PC1 主要由 GDP 和 Income 决定，我们可以把它命名为什么？载荷符号为正/负分别代表什么？

import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import load_iris
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA

# --- Parameters (user-tunable) ---
n_components = 2

# --- Load and standardize data ---
iris = load_iris()
X = iris.data
feature_names = iris.feature_names
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# --- PCA ---
pca = PCA(n_components=n_components)
X_pca = pca.fit_transform(X_scaled)

# --- Loadings (feature contributions to each PC) ---
loadings = pca.components_.T * np.sqrt(pca.explained_variance_)

# --- Biplot: scores + loadings ---
fig, ax = plt.subplots(figsize=(7, 6))
ax.scatter(X_pca[:, 0], X_pca[:, 1], alpha=0.5, edgecolor='k')
for i, name in enumerate(feature_names):
    ax.arrow(0, 0, loadings[i, 0]*2, loadings[i, 1]*2, color='red', head_width=0.1)
    ax.text(loadings[i, 0]*2.2, loadings[i, 1]*2.2, name, color='red')
ax.set_xlabel('PC1')
ax.set_ylabel('PC2')
ax.set_title('PCA Biplot: Scores and Loadings')
ax.axhline(0, color='gray', linestyle='--', alpha=0.5)
ax.axvline(0, color='gray', linestyle='--', alpha=0.5)
plt.tight_layout()
plt.show()

Block 7：Min-Max / Z-Score / Robust Scaling

问题：不同特征量纲差异巨大（IELTS 0-9，TOEFL PBT 300-677），直接用原始值会让大数值特征主导距离计算。

三种常用缩放：

  • Min-Max：映射到 [0,1]，适合 K-Means、PCA
  • Z-Score：均值为 0、标准差为 1，保留分布形状
  • Robust：基于四分位数，抗异常值

Block 10：Label Encoding vs One-Hot Encoding

模型只认数字，但原始数据常有类别。

两种编码：

  • Label Encoding：把有序类别映射为整数（Low=1, Medium=2, High=3）。树模型也能接受名义类别。
  • One-Hot Encoding：名义类别无顺序时，用 n-1 或 n 个 0/1 列表示。

启发问题：把“红、绿、蓝”编码成 0、1、2 会给线性模型带来什么错误假设？

import pandas as pd
from sklearn.preprocessing import LabelEncoder

# --- Parameters (user-tunable) ---
df = pd.DataFrame({
    'City': ['A', 'B', 'C', 'A', 'B'],
    'Size': ['Small', 'Medium', 'Large', 'Medium', 'Small'],
    'Passed': ['Yes', 'No', 'Yes', 'Yes', 'No']
})

# --- Ordinal label encoding for Size ---
size_order = {'Small': 1, 'Medium': 2, 'Large': 3}
df['Size_Label'] = df['Size'].map(size_order)

# --- Label encoding for tree-based models ---
le = LabelEncoder()
df['City_Label'] = le.fit_transform(df['City'])

# --- One-hot encoding for nominal City ---
df_onehot = pd.get_dummies(df['City'], prefix='City', drop_first=False)

df_out = pd.concat([df, df_onehot], axis=1)
print(df_out)

Block 12：缺失值检测与插补

缺失值处理：传感器故障、人工录入错误都会导致缺失。检测方法包括 isnull().sum()、直方图、描述统计。

常用策略：

  • 删除（缺失很少时）
  • 均值 / 中位数 / 众数填充
  • 前向填充（时间序列）
  • 线性插值（保留趋势）

Block 13：异常值检测与封顶

异常值：极端值会扭曲均值、标准差和回归结果。检测方法包括箱线图、Z-score、IQR。

处理策略：

  • 删除（确认是错误时）
  • 对数变换（缓解右偏）
  • 分位数封顶（capping）

IQR 法则：超出 [Q1 - 1.5·IQR, Q3 + 1.5·IQR] 视为异常。

启发问题：异常值一定是“错误”吗？在欺诈检测、疫情预警中，异常值反而是不是信号？

import numpy as np
import matplotlib.pyplot as plt

# --- Parameters (user-tunable) ---
np.random.seed(42)
data = np.concatenate([
    np.random.normal(100, 10, 100),
    [150, 155, 45]  # obvious outliers
])

# --- Z-score method ---
z_scores = np.abs((data - data.mean()) / data.std())
z_outliers = data[z_scores > 3]

# --- IQR method ---
Q1, Q3 = np.percentile(data, [25, 75])
IQR = Q3 - Q1
lower = Q1 - 1.5 * IQR
upper = Q3 + 1.5 * IQR
iqr_outliers = data[(data < lower) | (data > upper)]

# --- Capping at Q3 + 1.5 IQR ---
data_capped = np.clip(data, lower, upper)

# --- Visualization ---
fig, axes = plt.subplots(1, 2, figsize=(12, 4))
axes[0].boxplot(data, vert=False)
axes[0].set_title('Original Data with Outliers')
axes[1].boxplot(data_capped, vert=False)
axes[1].set_title('Data Capped at Q3 + 1.5 IQR')
plt.tight_layout()
plt.show()

print(f'Z-score outliers: {z_outliers}')
print(f'IQR outliers: {iqr_outliers}')

Block 15：公交能耗仿真

数据仿真：真实数据不可得、太复杂或敏感时，用规则 + 概率分布生成合成数据，测试模型鲁棒性、探索 what-if 场景。

公交能耗示例规则：

Base_Energy = Route_Length × 1.5 kWh/mile

Load_Factor = 1 + 0.3 × Passenger_Load

Daily_Energy = Base_Energy × Load_Factor × Weather_Factor

启发问题：2023A 蒲公英扩散为什么需要数据仿真？仿真参数（均值、方差、分布类型）应从哪来？

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt

# --- Parameters (user-tunable) ---
np.random.seed(42)
n_buses = 10
n_days = 365

# --- Define simulation rules ---
route_length = np.random.uniform(50, 150, n_buses)      # miles per day
passenger_load = np.random.uniform(0.2, 1.0, n_buses)   # fraction of capacity
weather_factor = np.random.choice([0.9, 1.0, 1.1], size=(n_buses, n_days))

# --- Simulate daily energy consumption ---
# Base_Energy = Route_Length * 1.5 kWh/mile
# Load_Factor = 1 + 0.3 * Passenger_Load
base_energy = route_length[:, None] * 1.5
load_factor = 1 + 0.3 * passenger_load[:, None]
daily_energy = base_energy * load_factor * weather_factor

# --- Summary ---
df = pd.DataFrame(daily_energy.T, columns=[f'Bus_{i+1}' for i in range(n_buses)])
print(df.head())
print(f"\nFleet daily mean: {daily_energy.mean():.2f} kWh")
print(f"Fleet daily std: {daily_energy.std():.2f} kWh")

# --- Visualization ---
fig, axes = plt.subplots(1, 2, figsize=(13, 4))
axes[0].plot(df.mean(axis=1), color='steelblue')
axes[0].set_title('Fleet Average Daily Energy Consumption')
axes[0].set_xlabel('Day')
axes[0].set_ylabel('kWh')
axes[1].hist(daily_energy.flatten(), bins=50, color='seagreen', edgecolor='white')
axes[1].set_title('Distribution of Daily Energy Use')
axes[1].set_xlabel('kWh')
plt.tight_layout()
