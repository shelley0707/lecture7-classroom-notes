1. EDA — Explore Hawaii Visitor Data

We model weekly visitor arrivals to Hawaii. Time series data has one dependent variable recorded over time, so we first visualize the raw series.

  • Time plot: reveals trend (long-term direction) and seasonality (repeating patterns).
  • Yearly seasonality: Hawaii tourism typically peaks in summer and around winter holidays.
  • Train-test split: reserve the most recent 20% of weeks for evaluation.
Takeaway: Always inspect the time plot before choosing a model. Trend + seasonality suggests decomposition or SARIMA. 
import warnings
warnings.filterwarnings('ignore')

import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns

# -----------------------------
# Generate synthetic weekly Hawaii visitor data
# -----------------------------
np.random.seed(42)
start_date = pd.to_datetime('2020-01-05')  # Sunday
n_weeks = 208  # 4 years of weekly data
dates = pd.date_range(start=start_date, periods=n_weeks, freq='W-SUN')

# Upward trend (tourism recovery/growth)
trend = np.linspace(150_000, 270_000, n_weeks)

# Yearly seasonality: peak in summer (weeks ~25-35) and winter holidays (weeks ~48-52)
week_of_year = np.array([d.weekofyear for d in dates])
seasonal = 35_000 * np.sin(2 * np.pi * week_of_year / 52) \
         + 18_000 * np.cos(2 * np.pi * week_of_year / 52)

# COVID-like shock in early 2020
shock = np.zeros(n_weeks)
shock[:20] = -60_000 * np.exp(-np.arange(20) / 5)

# Random noise
noise = np.random.normal(0, 7_000, n_weeks)

visitors = trend + seasonal + shock + noise
visitors = np.maximum(visitors, 50_000).astype(int)

df = pd.DataFrame({
    'date': dates,
    'visitors': visitors
})
df.set_index('date', inplace=True)
df.index.freq = 'W-SUN'

# Train-test split: last 20% as test
train_size = int(0.8 * len(df))
train = df.iloc[:train_size].copy()
test = df.iloc[train_size:].copy()

print(f"Total weeks: {len(df)}")
print(f"Train weeks: {len(train)}")
print(f"Test weeks:  {len(test)}")
print("\nFirst 5 rows:")
print(df.head())
print("\nBasic statistics:")
print(df.describe())

# Time plot
plt.figure(figsize=(12, 5))
plt.plot(train.index, train['visitors'], label='Train', color='steelblue')
plt.plot(test.index, test['visitors'], label='Test', color='coral')
plt.title('Weekly Visitor Arrivals to Hawaii')
plt.xlabel('Date')
plt.ylabel('Visitors')
plt.legend()
plt.grid(True, alpha=0.3)
plt.tight_layout()
plt.show()

2. Method 1 — Additive Decomposition

An additive model assumes the observed series is the sum of trend, seasonal, and residual components:

y(t) = Trend(t) + Seasonal(t) + Residual(t)

  • Trend: long-term movement in visitor numbers.
  • Seasonal: repeating yearly pattern (weekly period = 52).
  • Residual: what remains after removing trend and seasonality.
  • Forecast: extend the trend linearly and repeat the seasonal pattern to obtain predicted visitor counts immediately after decomposition.
Takeaway: Additive decomposition works when seasonal swings stay roughly constant over time. You can forecast right away by projecting trend + seasonality. 
from statsmodels.tsa.seasonal import seasonal_decompose

# -----------------------------
# Additive decomposition on training data
# -----------------------------
decomp = seasonal_decompose(
    train['visitors'],
    model='additive',
    period=52  # yearly seasonality for weekly data
)

# Plot decomposition
fig, axes = plt.subplots(4, 1, figsize=(12, 10), sharex=True)
decomp.observed.plot(ax=axes[0], title='Observed', color='steelblue')
decomp.trend.plot(ax=axes[1], title='Trend', color='darkgreen')
decomp.seasonal.plot(ax=axes[2], title='Seasonal', color='darkorange')
decomp.resid.plot(ax=axes[3], title='Residual', color='gray')
for ax in axes:
    ax.grid(True, alpha=0.3)
plt.suptitle('Additive Decomposition (Weekly, period=52)', y=1.01)
plt.tight_layout()
plt.show()

# Print component ranges
print(f"Trend range:    {decomp.trend.min():,.0f} to {decomp.trend.max():,.0f}")
print(f"Seasonal range: {decomp.seasonal.min():,.0f} to {decomp.seasonal.max():,.0f}")
print(f"Residual std:   {decomp.resid.std():,.0f}")

# -----------------------------
# Decomposition-based forecast
# -----------------------------
# Extend trend linearly from the observed trend component
last_trend = decomp.trend.dropna()
trend_slope = (last_trend.iloc[-1] - last_trend.iloc[0]) / (len(last_trend) - 1)
future_trend = [last_trend.iloc[-1] + trend_slope * (i + 1) for i in range(len(test))]

# Repeat the estimated seasonal pattern (period = 52)
seasonal_values = decomp.seasonal.values
future_seasonal = []
for i in range(len(test)):
    idx = (len(train) + i) % 52
    future_seasonal.append(seasonal_values[idx])

# Predicted visitor counts = extended trend + repeated seasonality
decomp_forecast = pd.Series(
    np.array(future_trend) + np.array(future_seasonal),
    index=test.index
)

print("\nDecomposition forecast (first 5 weeks of test set):")
print(decomp_forecast.head())
print(f"\nForecast range: {decomp_forecast.min():,.0f} to {decomp_forecast.max():,.0f}")

# Plot the decomposition forecast against train and test data
plt.figure(figsize=(12, 5))
plt.plot(train.index, train['visitors'], label='Train', color='steelblue')
plt.plot(test.index, test['visitors'], label='Test', color='coral')
plt.plot(decomp_forecast.index, decomp_forecast, label='Decomposition Forecast', color='purple', linestyle='--')
plt.title('Weekly Visitor Arrivals to Hawaii with Decomposition Forecast')
plt.xlabel('Date')
plt.ylabel('Visitors')
plt.legend()
plt.grid(True, alpha=0.3)
plt.tight_layout()
plt.show()

3. Stationarity Check

SARIMA requires the series to be stationary (constant mean, variance, and autocorrelation). We test with the Augmented Dickey-Fuller (ADF) test. The raw Hawaii visitor series has a clear upward trend, so we expect it to be non-stationary.

  • Null hypothesis: the series has a unit root (non-stationary).
  • p-value < 0.05: reject the null; series is stationary.
  • p-value ≥ 0.05: fail to reject the null; series is non-stationary (needs differencing).
  • Differencing: compute y'(t) = y(t) - y(t-1) to remove trend.
Takeaway: The raw series should be non-stationary because of trend. After first differencing it should become stationary. The differencing count becomes the d parameter. 
from statsmodels.tsa.stattools import adfuller

# -----------------------------
# ADF test helper
# -----------------------------
def adf_test(series, title=''):
    result = adfuller(series.dropna())
    print(f"\nADF Test: {title}")
    print(f"  ADF Statistic: {result[0]:.4f}")
    print(f"  p-value:       {result[1]:.4f}")
    print(f"  Critical 5%:   {result[4]['5%']:.4f}")
    if result[1] < 0.05:
        print("  => Stationary (reject unit root)")
    else:
        print("  => Non-stationary (fail to reject unit root)")

adf_test(train['visitors'], 'Original series')

# First-order differencing
train_diff = train['visitors'].diff().dropna()
adf_test(train_diff, 'First differenced series')

# Plot differenced series
plt.figure(figsize=(12, 4))
plt.plot(train_diff.index, train_diff, color='steelblue')
plt.title('First-Differenced Weekly Visitor Arrivals')
plt.xlabel('Date')
plt.ylabel('Change in visitors')
plt.grid(True, alpha=0.3)
plt.tight_layout()
plt.show()

4. ACF / PACF for SARIMA Order Selection

Autocorrelation (ACF) and partial autocorrelation (PACF) guide the choice of AR and MA orders. The plots below are drawn on the differenced series, because we already established it is stationary.

How to read ACF and PACF plots

  • Significance: spikes that cross the blue shaded confidence band are statistically significant.
  • PACF sharp cutoff: the last significant lag before the PACF drops to zero gives the AR(p) order. Example: significant spikes at lags 1 and 2 only → p = 2.
  • ACF sharp cutoff: the last significant lag before the ACF drops to zero gives the MA(q) order. Example: significant spike at lag 1 only → q = 1.
  • Gradual decay: if ACF or PACF tails off slowly, more differencing or higher-order terms may be needed.
  • Seasonal spikes: significant spikes at lag 52 (and 104, 156, …) indicate yearly seasonality. They suggest seasonal AR(P) or seasonal MA(Q) terms with period m = 52.
Takeaway: PACF cutoff → AR order; ACF cutoff → MA order; seasonal spikes at lag 52 → need seasonal terms. These plots give a starting point; refine orders with AIC/BIC grid search. 
from statsmodels.graphics.tsaplots import plot_acf, plot_pacf

# -----------------------------
# ACF and PACF on differenced training data
# -----------------------------
fig, axes = plt.subplots(1, 2, figsize=(14, 4))
plot_acf(train_diff, ax=axes[0], lags=60, title='ACF (Differenced)')
plot_pacf(train_diff, ax=axes[1], lags=60, title='PACF (Differenced)', method='ywm')
plt.tight_layout()
plt.show()

print("Look for:")
print("- Significant spikes in PACF -> AR order p")
print("- Significant spikes in ACF  -> MA order q")
print("- Spikes near lag 52 -> seasonal component")

5. Method 2 — SARIMA Modeling

SARIMA extends ARIMA with a seasonal part. A model SARIMA(p,d,q)(P,D,Q)m has:

  • (p,d,q): non-seasonal AR, differencing, MA orders.
  • (P,D,Q): seasonal AR, seasonal differencing, seasonal MA orders.
  • m: seasonal period (52 for weekly yearly seasonality).

We use AIC to choose among candidate models, then fit the best one.

Takeaway: SARIMA captures both short-term dynamics and recurring yearly patterns in one model. 
from statsmodels.tsa.statespace.sarimax import SARIMAX

# -----------------------------
# Small candidate grid for SARIMA orders
# -----------------------------
# Weekly data with yearly seasonality -> seasonal period m = 52
seasonal_period = 52
candidates = [
    ((0, 1, 1), (0, 1, 1, seasonal_period)),
    ((1, 1, 0), (1, 1, 0, seasonal_period)),
    ((1, 1, 1), (1, 1, 1, seasonal_period)),
    ((1, 1, 2), (1, 1, 1, seasonal_period)),
]

best_aic = np.inf
best_order = None
best_seasonal_order = None

print("Searching SARIMA orders (this may take a minute)...")
for param, seasonal_param in candidates:
    try:
        model = SARIMAX(
            train['visitors'],
            order=param,
            seasonal_order=seasonal_param,
            enforce_stationarity=False,
            enforce_invertibility=False
        )
        results = model.fit(disp=False)
        print(f"SARIMA{param}{seasonal_param} -> AIC: {results.aic:.2f}")
        if results.aic < best_aic:
            best_aic = results.aic
            best_order = param
            best_seasonal_order = seasonal_param
    except Exception as e:
        print(f"SARIMA{param}{seasonal_param} failed: {e}")
        continue

print(f"\nBest SARIMA order:        {best_order}")
print(f"Best seasonal order:      {best_seasonal_order}")
print(f"Best AIC:                 {best_aic:.2f}")

# Fit best model
final_sarima = SARIMAX(
    train['visitors'],
    order=best_order,
    seasonal_order=best_seasonal_order,
    enforce_stationarity=False,
    enforce_invertibility=False
)
sarima_result = final_sarima.fit(disp=False)
print(sarima_result.summary())

6. Forecast & Model Comparison

We forecast the test period with both methods and compare errors.

  • Decomposition forecast: extend the trend and repeat the seasonal pattern, then add them.
  • SARIMA forecast: use the fitted model's get_forecast() method.
  • Metrics: MAE, RMSE, MAPE.
Takeaway: SARIMA usually outperforms simple decomposition on short-term forecasts because it models autocorrelation in the residuals. 
from sklearn.metrics import mean_absolute_error, mean_squared_error

# -----------------------------
# Method 1: Decomposition forecast
# -----------------------------
# Extend trend linearly
last_trend = decomp.trend.dropna()
trend_slope = (last_trend.iloc[-1] - last_trend.iloc[0]) / (len(last_trend) - 1)
future_trend = [last_trend.iloc[-1] + trend_slope * (i + 1) for i in range(len(test))]

# Repeat seasonal pattern
seasonal_values = decomp.seasonal.values
future_seasonal = []
for i in range(len(test)):
    idx = (len(train) + i) % 52
    future_seasonal.append(seasonal_values[idx])

decomp_forecast = pd.Series(
    np.array(future_trend) + np.array(future_seasonal),
    index=test.index
)

# -----------------------------
# Method 2: SARIMA forecast
# -----------------------------
sarima_forecast = sarima_result.get_forecast(steps=len(test))
sarima_mean = sarima_forecast.predicted_mean

# -----------------------------
# Evaluation metrics
# -----------------------------
def mape(y_true, y_pred):
    return np.mean(np.abs((y_true - y_pred) / y_true)) * 100

metrics = pd.DataFrame({
    'Model': ['Additive Decomposition', 'SARIMA'],
    'MAE': [
        mean_absolute_error(test['visitors'], decomp_forecast),
        mean_absolute_error(test['visitors'], sarima_mean)
    ],
    'RMSE': [
        np.sqrt(mean_squared_error(test['visitors'], decomp_forecast)),
        np.sqrt(mean_squared_error(test['visitors'], sarima_mean))
    ],
    'MAPE (%)': [
        mape(test['visitors'], decomp_forecast),
        mape(test['visitors'], sarima_mean)
    ]
})

print(metrics.to_string(index=False))

# Plot forecasts
plt.figure(figsize=(12, 6))
plt.plot(train.index, train['visitors'], label='Train', color='steelblue')
plt.plot(test.index, test['visitors'], label='Test', color='black', linewidth=2)
plt.plot(test.index, decomp_forecast, label='Decomposition Forecast', color='green', linestyle='--')
plt.plot(test.index, sarima_mean, label='SARIMA Forecast', color='purple')
plt.title('Hawaii Weekly Visitor Arrivals — Forecast Comparison')
plt.xlabel('Date')
plt.ylabel('Visitors')
plt.legend()
plt.grid(True, alpha=0.3)
plt.tight_layout()
plt.show()

7. Residual Diagnostics

Good forecasts should leave white-noise residuals: zero mean, constant variance, and no remaining autocorrelation.

  • Residual time plot: check for patterns or changing variance.
  • ACF of residuals: should have no significant spikes.
  • Ljung-Box test: null hypothesis is that residuals are white noise. p-value > 0.05 means we fail to reject (good).
Takeaway: If residuals are not white noise, the model has missed patterns and needs more tuning. 
from statsmodels.stats.diagnostic import acorr_ljungbox
from statsmodels.graphics.tsaplots import plot_acf

# -----------------------------
# SARIMA residual diagnostics
# -----------------------------
residuals = sarima_result.resid

fig, axes = plt.subplots(2, 1, figsize=(12, 8))
residuals.plot(ax=axes[0], title='SARIMA Residuals', color='steelblue')
axes[0].axhline(y=0, color='r', linestyle='--')
axes[0].grid(True, alpha=0.3)

plot_acf(residuals, ax=axes[1], lags=60, title='ACF of Residuals')
axes[1].grid(True, alpha=0.3)
plt.tight_layout()
plt.show()

# Ljung-Box test
lb_test = acorr_ljungbox(residuals, lags=[10, 20, 30], return_df=True)
print("\nLjung-Box test on residuals:")
print(lb_test)

p_val_10 = lb_test.loc[10, 'lb_pvalue'] if 10 in lb_test.index else lb_test.iloc[0]['lb_pvalue']
if p_val_10 > 0.05:
    print("\n=> Residuals look like white noise (good).")
else:
    print("\n=> Residuals still show autocorrelation (model can be improved).")

8. Summary

Time series forecasting workflow for Hawaii visitor arrivals.

  • Method 1 — Additive Decomposition: intuitive, separates trend and seasonality, good for understanding components.
  • Method 2 — SARIMA: models autocorrelation directly, usually better for short-term forecasts.
  • Diagnostics: always check residuals for white noise.
Takeaway: Start with decomposition to understand the data; use SARIMA when you need a more accurate, self-correcting forecast.
