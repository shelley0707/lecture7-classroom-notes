Simulation & Stochastic Models

Classroom Notes: Embracing Uncertainty — From Random Sampling to Emergent Behavior

Monte Carlo Agent-Based Markov Chains Parameter Insight

A deterministic model gives the same output for the same input (a cannonball's trajectory). A stochastic model embraces uncertainty: it does not predict one fixed answer but the distribution of possible answers (a 60% chance of rain). This lecture builds three of the most useful stochastic tools. To keep every model as precise as a linear program, each example below is stated with the same four-part contract:

State & Random Variables
what the simulation tracks and what is drawn at random — the stochastic analog of decision variables.
Model Rules / Distributions
the equations, update rules, or probability laws every variable must obey — the analog of constraints.
Output Metric
the quantity we estimate from many runs — the analog of the objective function.
Parameters (assumed)
the fixed numbers we feed in and later vary to gain insight.

Part A  ·  Monte Carlo Simulation

Core idea (top-down): no complicated formula needed. Define the probability distribution for each uncertain input, then let a computer "roll the dice" thousands of times. The frequency of outcomes across all trials approximates the true probability — using quantitative change (many trials) to produce qualitative insight (the answer). The four steps: (1) define the problem, (2) build a probability model, (3) run many random trials, (4) analyze the results.

Monte Carlo

Example 1 — Estimating π by Throwing Darts

Scenario

Throw darts blindly at a 2×2 square with an inscribed circle of radius 1. The fraction landing inside the circle equals the ratio of areas, π/4. Counting hits therefore estimates π — the "hello world" of Monte Carlo.

State & Random Variables

  • (x, y) = coordinates of one dart, each drawn at random
  • hit = 1 if the dart lands inside the circle, else 0

Model Rules / Distributions

  • x ~ Uniform(-1, 1), y ~ Uniform(-1, 1) — every spot equally likely
  • inside the circle when x² + y² ≤ 1

Output Metric

π ≈ 4 × (darts inside circle / total darts)

Parameters (assumed)

  • Number of darts (trials): swept over 10 to 100,000
Python Demo 
import numpy as np
import matplotlib.pyplot as plt

np.random.seed(0)
N = 100000
x = np.random.uniform(-1, 1, N)
y = np.random.uniform(-1, 1, N)
inside = x**2 + y**2 <= 1

pi_estimate = 4 * inside.mean()
print("Estimate of pi with {} darts: {:.5f}".format(N, pi_estimate))

# --- Convergence: accuracy grows with the number of trials ---
trials = np.logspace(1, 5, 30).astype(int)
estimates = [4 * (np.random.uniform(-1, 1, n)**2 + np.random.uniform(-1, 1, n)**2 <= 1).mean()
             for n in trials]

fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(13, 5))
ax1.scatter(x[:3000][inside[:3000]], y[:3000][inside[:3000]], s=4, color="teal")
ax1.scatter(x[:3000][~inside[:3000]], y[:3000][~inside[:3000]], s=4, color="lightcoral")
ax1.set_aspect("equal"); ax1.set_title("Darts: inside vs outside the circle")
ax2.semilogx(trials, estimates, marker="o")
ax2.axhline(np.pi, color="red", ls="--", label="true pi")
ax2.set_xlabel("Number of trials"); ax2.set_ylabel("Estimate")
ax2.set_title("Law of Large Numbers: convergence to pi"); ax2.legend()
plt.tight_layout(); plt.show()

Gaining insight by changing parameters: the only knob is the trial count N. Watch the estimate jitter wildly for small N and tighten around π as N grows — this is the Law of Large Numbers in action. The error shrinks like 1/√N, so to gain one more decimal place of accuracy you need about 100× more darts. Ask yourself: why does quadrupling N only halve the error, and what does that imply about the cost of a "very precise" Monte Carlo answer?

Monte Carlo

Example 2 — The Wandering Leaf

Scenario

A leaf detaches 5 m above the ground and drifts down at a steady 1 m/s. A gusty horizontal wind pushes it sideways with a random speed. Where does it land? A single answer is meaningless — we want the distribution of landing spots so a groundskeeper knows where leaves tend to gather.

State & Random Variables

  • V = horizontal wind speed (m/s), the random input
  • X = final horizontal landing position (m), the output of each trial

Model Rules / Distributions

  • V ~ Normal(μ = 5, σ = 3) — natural phenomena cluster around a mean
  • fall time t = height / descent speed = 5 / 1 = 5 s (deterministic)
  • landing position X = V × t (physics rule linking random input to output)

Output Metric

The full distribution of X: its mean, spread, and the probability the leaf drifts past a chosen distance (e.g. P(X > 20 m)).

Parameters (assumed)

  • Initial height H = 5 m, descent speed = 1 m/s
  • Wind mean μ = 5 m/s, wind std σ = 3 m/s
  • Trials = 20,000
Python Demo 
import numpy as np
import matplotlib.pyplot as plt

np.random.seed(1)
H, descent = 5.0, 1.0        # height (m), vertical speed (m/s)
t_fall = H / descent         # 5 seconds in the air
N = 20000

# Compare how the SPREAD of wind changes where leaves pile up
plt.figure(figsize=(10, 5))
for sigma in [1, 3, 6]:
    V = np.random.normal(5, sigma, N)   # random wind speed
    X = V * t_fall                      # landing position
    plt.hist(X, bins=60, alpha=0.5, density=True,
             label="wind std = {}  ->  P(X>20m) = {:.2f}".format(sigma, np.mean(X > 20)))

plt.axvline(25, color="black", ls="--", label="mean landing = 25 m")
plt.xlabel("Horizontal landing position X (m)")
plt.ylabel("Probability density")
plt.title("Where the Leaf Lands: more wind variability = wider spread")
plt.legend(); plt.grid(True, alpha=0.3)
plt.tight_layout(); plt.show()

V = np.random.normal(5, 3, N)
X = V * t_fall
print("Mean landing: {:.1f} m".format(X.mean()))
print("Std of landing: {:.1f} m".format(X.std()))
print("P(leaf drifts past 20 m): {:.1%}".format(np.mean(X > 20)))

Gaining insight by changing parameters: the mean landing spot (25 m) is fixed by the average wind, but the risk of a leaf drifting far depends entirely on σ. Larger σ flattens and widens the histogram, raising P(X>20m). This separates the expected outcome from its uncertainty — the whole point of stochastic modeling. Then think deeper: what if the initial height were also random, or the descent were not a straight line? Each added random variable is one more input to sample, and the Monte Carlo recipe never changes.

Extension: A 2-D Wind Field & a Bird's-Eye Map of 10,000 Leaves

The 1-D model only let the leaf drift left or right. Reality is 2-D: wind has both a speed and a direction, and leaves drop from a range of heights. We promote all three to random variables, place the tree at the origin (0, 0), and simulate where 10,000 leaves settle on the ground — then look straight down from above.

How to define a random variable with a given distribution

The pattern is always np.random.<distribution>(parameters, size=N). Pick the distribution that matches the nature of the quantity:

  • Normal(μ, σ) — np.random.normal(mu, sigma, N) — natural quantities that cluster around an average (wind speed, direction).
  • Uniform(a, b) — np.random.uniform(a, b, N) — any value in a range equally likely (drop height between two branches).
  • Exponential(scale) — np.random.exponential(scale, N) — waiting times or "mostly small, occasionally large" gusts.
  • Poisson(λ) — np.random.poisson(lam, N) — counts of events in a fixed window.

State & Random Variables (now three)

  • wind_speed = how fast the wind pushes (m/s)
  • wind_angle = which compass direction the wind blows toward (radians)
  • height = the branch height the leaf falls from (m)
  • (x, y) = the leaf's final position on the ground, seen from above

Model Rules / Distributions

  • wind_speed ~ Normal(μ=5, σ=2), clipped at 0 (speed can't be negative)
  • wind_angle ~ Normal(0, 0.5) — a prevailing wind blowing east, with some wobble
  • height ~ Uniform(4, 6) — leaves hang between 4 m and 6 m
  • fall time t = height / descent; drift distance r = wind_speed × t
  • convert polar drift to map coordinates: x = r·cos(angle), y = r·sin(angle)

Output Metric

The 2-D landing map of 10,000 leaves: where they pile up, the mean drift distance, and the probability a leaf lands more than 15 m from the trunk.

Python Demo 
import numpy as np
import matplotlib.pyplot as plt

np.random.seed(7)
N = 10000                       # simulate 10,000 leaves
descent = 1.0                   # vertical fall speed (m/s), fixed

# ---------------------------------------------------------------
# Define EACH random variable with its own distribution.
# Template:  variable = np.random.(params, size=N)
# ---------------------------------------------------------------
wind_speed = np.random.normal(5, 2, N)      # Normal: clusters near 5 m/s
wind_speed = np.clip(wind_speed, 0, None)   # a speed cannot be negative
wind_angle = np.random.normal(0.0, 0.5, N)  # Normal: prevailing wind blows east (0 rad)
height     = np.random.uniform(4, 6, N)     # Uniform: any branch 4-6 m is equally likely

# ---------------------------------------------------------------
# Physics: turn the random inputs into a landing position (x, y)
# ---------------------------------------------------------------
t_fall = height / descent          # time in the air
r      = wind_speed * t_fall       # how far the wind carries it (radial)
x      = r * np.cos(wind_angle)    # east-west landing coordinate
y      = r * np.sin(wind_angle)    # north-south landing coordinate

# ---------------------------------------------------------------
# Bird's-eye view: look straight down at the ground
# ---------------------------------------------------------------
fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(14, 6))

ax1.scatter(x, y, s=3, alpha=0.15, color="saddlebrown")
ax1.plot(0, 0, marker="*", color="green", markersize=18, label="tree trunk")
ax1.set_aspect("equal"); ax1.set_xlabel("East-West (m)"); ax1.set_ylabel("North-South (m)")
ax1.set_title("Bird's-eye view: where 10,000 leaves land"); ax1.legend()

hb = ax2.hexbin(x, y, gridsize=45, cmap="YlOrBr", mincnt=1)
ax2.plot(0, 0, marker="*", color="green", markersize=18)
ax2.set_aspect("equal"); ax2.set_xlabel("East-West (m)"); ax2.set_ylabel("North-South (m)")
ax2.set_title("Leaf density (where they pile up)")
fig.colorbar(hb, ax=ax2, label="leaves per cell")
plt.tight_layout(); plt.show()

dist = np.sqrt(x**2 + y**2)
print("Mean drift distance from trunk: {:.1f} m".format(dist.mean()))
print("Leaves settle around x = {:.1f} m (downwind), y ~ {:.1f} m".format(x.mean(), y.mean()))
print("P(leaf lands more than 15 m away): {:.1%}".format(np.mean(dist > 15)))

Gaining insight by changing parameters: the map is a cigar shape stretched east — the prevailing wind direction sets where the pile forms, while the angle spread σ sets how much it fans out sideways. Turn the angle into np.random.uniform(0, 2*np.pi, N) and the leaves scatter in a full ring around the trunk; shrink the angle's σ toward 0 and they collapse back to the 1-D line from before. This is the key lesson: each extra random variable adds a new dimension of realism, but the recipe never changes — define a distribution, sample N times, push the samples through the physics, and read the resulting map. If you were a city planner deciding where to place leaf-collection bins, which single parameter would you measure most carefully?

Monte Carlo

Example 3 — Product Launch Timeline & Risk

Scenario

A startup must ship its flagship product. Eight tasks (A–H) run partly in parallel and partly in sequence, but each task's duration is uncertain. The CEO wants the probability of launching within the critical 30-day window and needs to know which task is the real bottleneck.

State & Random Variables

  • d_A … d_H = the random duration of each of the eight tasks
  • finish_i = the time task i completes, given its predecessors
  • T = total project time = finish time of the final task H

Model Rules / Distributions

  • Durations: A~N(10,2), B~N(7,1.5), C~N(8,2), D~N(6,1), E~U(5,10), F~N(12,3), G~U(3,7), H=1
  • Dependencies: C after A; D after B; E after C & D; G after A & B; H after E, F & G
  • A task starts only when all its predecessors finish: finish = max(predecessor finishes) + duration

Output Metric

Distribution of total time T; P(T ≤ 30 days); and the correlation of each task with T to reveal the bottleneck.

Parameters (assumed)

  • Task means / spreads as above; launch deadline = 30 days
  • Trials = 10,000
Python Demo 
import numpy as np
import matplotlib.pyplot as plt

np.random.seed(2)
N = 10000
durations = {k: np.zeros(N) for k in "ABCDEFG"}
durations["A"] = np.random.normal(10, 2, N)
durations["B"] = np.random.normal(7, 1.5, N)
durations["C"] = np.random.normal(8, 2, N)
durations["D"] = np.random.normal(6, 1, N)
durations["E"] = np.random.uniform(5, 10, N)
durations["F"] = np.random.normal(12, 3, N)
durations["G"] = np.random.uniform(3, 7, N)

# Finish times follow the dependency network (a task waits for its predecessors)
fA, fB = durations["A"], durations["B"]
fC = fA + durations["C"]
fD = fB + durations["D"]
fE = np.maximum(fC, fD) + durations["E"]
fG = np.maximum(fA, fB) + durations["G"]
fF = durations["F"]
T = np.maximum.reduce([fE, fF, fG]) + 1   # + H (constant 1 day)

print("Mean completion time: {:.1f} days".format(T.mean()))
print("P(launch within 30 days): {:.1%}".format(np.mean(T <= 30)))
print("90th percentile (plan-for-this): {:.1f} days".format(np.percentile(T, 90)))

# Which task drives the total? Correlate each duration with T.
print("\nBottleneck ranking (correlation of task duration with total time):")
for k in sorted(durations, key=lambda k: -np.corrcoef(durations[k], T)[0, 1]):
    print("  Task {}: {:+.2f}".format(k, np.corrcoef(durations[k], T)[0, 1]))

fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(13, 5))
ax1.hist(T, bins=50, color="teal", alpha=0.7)
ax1.axvline(30, color="red", ls="--", label="30-day deadline")
ax1.set_xlabel("Total project time (days)"); ax1.set_ylabel("Frequency")
ax1.set_title("Distribution of Launch Time"); ax1.legend()
corrs = {k: np.corrcoef(durations[k], T)[0, 1] for k in durations}
ax2.barh(list(corrs), list(corrs.values()), color="slateblue")
ax2.set_xlabel("Correlation with total time")
ax2.set_title("Bottleneck: which task controls the schedule?")
plt.tight_layout(); plt.show()

Gaining insight by changing parameters: a single-point estimate ("about 27 days") hides the risk — the simulation shows only ~87% of runs finish by day 30. The correlation chart is the payoff: the task most correlated with T is the true bottleneck, so shortening it (not a random task) buys the most schedule safety. Try halving the std of the top task versus the bottom task and re-run: which actually raises P(T≤30)? This is how a modeler turns "work harder" into "work on the right thing."

Part B  ·  Agent-Based Simulation

Core idea (bottom-up): instead of one equation for the whole system, define simple rules for individual agents and let global patterns emerge from their local interactions — like a stadium "Mexican wave" that nobody choreographs. The three pillars are Agents (autonomous individuals with their own state), the Environment (the stage and its resources), and Rules (how agents act and interact). This is the natural counterpart to the Lotka-Volterra differential equations: here each animal is its own decision-maker.

Agent-Based

Example 4 — The Wolf-Sheep-Grass Ecosystem

Scenario

A grid world holds three kinds of agents: grass (regrows), sheep (eat grass), and wolves (eat sheep). Nobody programs the population curves — the predator-prey oscillations emerge from four simple local rules. This is the agent-level story behind the smooth Lotka-Volterra curves.

Agents & State (the "variables")

  • Each sheep & wolf: position (x, y) on the grid and an energy level
  • Environment: a grid cell is either grass = 1 (grown) or 0 (eaten)

Rules (the "constraints")

  • Move: every agent takes one random step and loses 1 energy (living costs energy)
  • Eat: a sheep on a grass cell eats it (+energy, grass→0); a wolf on a sheep's cell eats it (+energy)
  • Reproduce: an agent with energy above a threshold splits, sharing energy with its offspring
  • Die: any agent whose energy hits 0 is removed; grass regrows at a fixed probability

Output Metric

The three population counts (grass, sheep, wolves) over time, and whether they settle into stable oscillations or one species collapses.

Parameters (assumed)

  • Grid 30×30; start 60 sheep, 20 wolves
  • Energy from grass = +4, from sheep = +20; grass regrowth prob = 0.03
  • Reproduce thresholds: sheep > 15, wolf > 40; run for 200 steps
Python Demo 
import numpy as np
import matplotlib.pyplot as plt

np.random.seed(3)

class Ecosystem:
    def __init__(self, size=30, n_sheep=60, n_wolves=20, regrow=0.03):
        self.size, self.regrow = size, regrow
        rand_pos = lambda: [np.random.randint(0, size), np.random.randint(0, size)]
        self.sheep  = [rand_pos() + [5]  for _ in range(n_sheep)]   # [x, y, energy]
        self.wolves = [rand_pos() + [20] for _ in range(n_wolves)]
        self.grass  = np.random.randint(0, 2, (size, size))

    def _move(self, a):
        a[0] = (a[0] + np.random.randint(-1, 2)) % self.size
        a[1] = (a[1] + np.random.randint(-1, 2)) % self.size

    def step(self):
        # Environment: grass regrows on empty cells
        regrow = (self.grass == 0) & (np.random.random(self.grass.shape) < self.regrow)
        self.grass = np.where(regrow, 1, self.grass)
        # Sheep: move, graze, reproduce, die
        survivors = []
        for s in self.sheep:
            self._move(s); s[2] -= 1
            if self.grass[s[0], s[1]] == 1:
                self.grass[s[0], s[1]] = 0; s[2] += 4
            if s[2] > 15:
                s[2] //= 2; survivors.append([s[0], s[1], s[2]])
            if s[2] > 0: survivors.append(s)
        self.sheep = survivors
        # Wolves: move, hunt, reproduce, die
        survivors = []
        for w in self.wolves:
            self._move(w); w[2] -= 1
            for s in self.sheep:
                if s[0] == w[0] and s[1] == w[1]:
                    w[2] += 20; self.sheep.remove(s); break
            if w[2] > 40:
                w[2] //= 2; survivors.append([w[0], w[1], w[2]])
            if w[2] > 0: survivors.append(w)
        self.wolves = survivors

eco = Ecosystem()
sheep_hist, wolf_hist, grass_hist = [], [], []
for _ in range(200):
    eco.step()
    sheep_hist.append(len(eco.sheep))
    wolf_hist.append(len(eco.wolves))
    grass_hist.append(int(eco.grass.sum()))

plt.figure(figsize=(11, 5))
plt.plot(sheep_hist, label="Sheep", color="seagreen")
plt.plot(wolf_hist, label="Wolves", color="firebrick")
plt.plot(np.array(grass_hist) / 10, label="Grass / 10", color="goldenrod", alpha=0.6)
plt.xlabel("Time step"); plt.ylabel("Population")
plt.title("Emergent Predator-Prey Cycles from Simple Local Rules")
plt.legend(); plt.grid(True, alpha=0.3)
plt.tight_layout(); plt.show()

print("Final -> sheep: {}, wolves: {}, grass cells: {}".format(
    sheep_hist[-1], wolf_hist[-1], grass_hist[-1]))

Gaining insight by changing parameters: no equation dictated the cycles — they emerged. Now experiment like an ecologist. Raise the grass regrowth rate: do sheep explode and then crash the wolves, or does the system find a new balance? Increase the wolves' energy gain (hunting efficiency): does a "better" predator win, or does it eat itself into starvation by wiping out its own food? What happens if wolves start too few? These are the deck's discussion questions, and each is answered by changing one number and watching the emergent result — the bottom-up mirror of the sensitivity analysis you did on the Lotka-Volterra equations.

Part C  ·  Markov Chains

Core idea (memoryless): the future depends only on the present state, not the path taken to get there — like deciding your next chess move from the current board alone. Formally P(Xₙ₁ | Xₙ, …, X₁) = P(Xₙ₁ | Xₙ). A single transition matrix P encodes all behavior, and repeatedly applying it reveals the long-run stationary distribution π = πP — where the system settles regardless of where it started.

Markov Chain

Example 5 — Bike-Sharing Station Balance

Scenario

A district has three bike stations A, B, C. Each trip a bike takes moves it between stations with fixed, historically-measured probabilities. Operators want to know the long-run distribution of bikes — which stations fill up and which run dry — so they can rebalance efficiently.

State Space (the "variables")

  • State = the station a bike currently sits at: S = {A, B, C}
  • πₜ = distribution vector = fraction of bikes at each station at time t

Model Rules (the "constraints")

  • Transition matrix P: entry P[i][j] = probability a bike at station i moves to j
  • Each row sums to 1 (a bike must go somewhere)
  • Markov property: next station depends only on the current station
  • Evolution rule: πₜ₊₁ = πₜ · P

Output Metric

The stationary distribution π satisfying π = πP — the long-run share of bikes per station, independent of the starting arrangement.

Parameters (assumed)

  • P = [[0.6, 0.3, 0.1], [0.2, 0.5, 0.3], [0.3, 0.2, 0.5]]
  • Start: all bikes at station A; iterate 30 steps
Python Demo 
import numpy as np
import matplotlib.pyplot as plt

# Transition matrix: row i -> probabilities of moving to A, B, C
P = np.array([[0.6, 0.3, 0.1],
              [0.2, 0.5, 0.3],
              [0.3, 0.2, 0.5]])
stations = ["A", "B", "C"]

# --- Iterate the evolution rule pi_{t+1} = pi_t * P ---
pi = np.array([1.0, 0.0, 0.0])   # start: every bike at A
history = [pi]
for _ in range(30):
    pi = pi @ P
    history.append(pi)
history = np.array(history)

# --- Verify with the eigenvector of P^T for eigenvalue 1 ---
vals, vecs = np.linalg.eig(P.T)
stationary = np.real(vecs[:, np.argmin(np.abs(vals - 1))])
stationary /= stationary.sum()

for s, p in zip(stations, stationary):
    print("Station {}: long-run share = {:.1%}".format(s, p))

plt.figure(figsize=(10, 5))
for i, s in enumerate(stations):
    plt.plot(history[:, i], marker="o", label="Station " + s)
    plt.axhline(stationary[i], color="gray", ls=":", alpha=0.5)
plt.xlabel("Time step"); plt.ylabel("Share of bikes")
plt.title("Convergence to the Stationary Distribution (starts all at A)")
plt.legend(); plt.grid(True, alpha=0.3)
plt.tight_layout(); plt.show()

Gaining insight by changing parameters: notice the curves forget their start — begin with every bike at A and they still settle to the same 38% / 34% / 28% split. That is the stationary distribution, and it tells operators station A quietly accumulates bikes while C runs dry, so trucks should carry bikes C→A each night. Now change one row of P (say users increasingly ride to C) and re-run: how does the steady state shift, and how many steps until it re-stabilizes? Try making a station "sticky" (raise its self-transition): does the whole network slow to equilibrium? This is exactly how Google's PageRank ranks web pages — states are pages, transitions are links, and the stationary distribution is importance.

Choosing Your Stochastic Tool

Model Worldview Best when… Key output
Monte Carlo Top-down: sample the whole system inputs are uncertain and you want the distribution / risk of an outcome probability & percentiles of a result
Agent-Based Bottom-up: rules for individuals system behavior emerges from many interacting, heterogeneous actors emergent global patterns over time
Markov Chain Memoryless state transitions the next state depends only on the current one, with fixed probabilities long-run stationary distribution
Expert's synthesis: every stochastic model answers "what usually happens, and how uncertain is it?" rather than "what is the single answer?" The modeling skill is matching the tool to the question: use Monte Carlo to quantify risk, agent-based models to explain emergence, and Markov chains to find long-run equilibria. In all three, insight comes not from a single run but from changing a parameter and watching how the distribution moves.
