0. 仿真模型是什么？一个统一的四步框架

确定性模型对同样的输入永远给出同样的输出；仿真模型则把不确定性直接写进模型里，回答的是“结果会怎么分布、风险有多大”。

无论 Monte Carlo、Agent-Based 还是概率优化，构造一个仿真模型都可以归为四步：

  1. 状态与随机变量：模型追踪什么？哪些是随机抽取的？
  2. 规则 / 分布：变量服从什么分布？agent 按什么规则更新？
  3. 输出指标：我们想估计的数值是什么？
  4. 参数与假设：哪些数是暂时固定的？改变它们会产生什么洞察？
探究假设的过程

仿真的价值不在于“跑一次”，而在于改变参数再跑。每次改动一个假设（例如把到达率 λ 变大、把传播概率降低、把冷却速度加快），观察输出指标如何移动，就能判断哪些假设对结论最敏感。

takeaway：好的仿真模型 = 清晰的假设 + 可重复的代码 + 对参数的系统性扫描。 
import numpy as np
import matplotlib.pyplot as plt

# 通用 Monte Carlo 骨架：也适用于绝大多数仿真模型的“核心循环”
def simulate_once(params):
    """单次仿真：根据参数生成随机输入并计算输出。"""
    x = np.random.normal(params['mu'], params['sigma'])
    return max(0, x)            # 例如：风速不能为负

def run_simulation(params, n_runs=10000):
    """重复 n_runs 次，返回输出分布。"""
    results = [simulate_once(params) for _ in range(n_runs)]
    return np.array(results)

params = {'mu': 5, 'sigma': 2}
out = run_simulation(params)
print(f"Mean={out.mean():.2f}, Std={out.std():.2f}, P>7={np.mean(out>7):.2%}")

1. Monte Carlo：弹珠抽样与大数定律

袋子里有 10% 红、30% 黄、60% 蓝弹珠。我们无法一次性清点整袋，于是不断随机抽取，用样本比例逼近真实比例。这是 Monte Carlo 最直观的形态。

Block 1：构建总体并单次抽样

状态与随机变量：每颗弹珠的颜色是离散的随机变量；总体比例已知，样本比例未知。

模型规则：

  • 总体：red=0.10, yellow=0.30, blue=0.60
  • 每次从袋中不放回地抽取 n 颗
  • 计算样本中各颜色占比

假设：袋子足够大（100 000 颗），单次抽样的 n 远小于总体，因此可近似看作独立同分布抽样。

import numpy as np
from collections import Counter

np.random.seed(0)
# 构建总体
total = 100000
marbles = (
    ['red'] * int(total * 0.10) +
    ['yellow'] * int(total * 0.30) +
    ['blue'] * int(total * 0.60)
)
np.random.shuffle(marbles)

def draw_and_report(n):
    drawn = np.random.choice(marbles, size=n, replace=False)
    counts = Counter(drawn)
    for color, true_p in [('red', 10), ('yellow', 30), ('blue', 60)]:
        pct = counts.get(color, 0) / n * 100
        print(f"{color:6}: {counts.get(color, 0):4d} ({pct:5.2f}%)  true={true_p}%")

print("--- Draw n=30 ---")
draw_and_report(30)
print("\n--- Draw n=5000 ---")
draw_and_report(5000)

Block 2：样本比例随抽样量收敛

输出指标：不同抽样量 n 下，各颜色样本比例与真实比例的差距。

探究假设：如果把总体中的蓝色比例从 60% 改成 90%，收敛曲线会更快还是更慢？如果袋子里只有 100 颗弹珠，“不放回”会带来什么偏差？

import numpy as np
import matplotlib.pyplot as plt

np.random.seed(1)
sizes = [10, 20, 50, 100, 200, 500, 1000, 2000, 5000]
red_err, blue_err = [], []

for n in sizes:
    drawn = np.random.choice(marbles, size=n, replace=False)
    red_err.append(abs((drawn == 'red').mean() - 0.10))
    blue_err.append(abs((drawn == 'blue').mean() - 0.60))

fig, ax = plt.subplots(figsize=(8, 4))
ax.plot(sizes, red_err, marker='o', label='|red - 10%|')

ax.set_xscale('log'); ax.set_yscale('log')
ax.set_xlabel('Sample size n'); ax.set_ylabel('Absolute error')
ax.set_title('Sample proportion converges to true proportion')
ax.legend(); ax.grid(True, alpha=0.3)
plt.tight_layout(); plt.show()

Block 3：用二项分布刻画“抽到蓝球”

模型规则：每次只关心“是否抽到蓝球”，则 n 次独立抽取中蓝球数量 X ~ Binomial(n, p=0.6)。

输出指标：模拟频率与理论概率质量函数 PMF 的对比。

假设再探：当 n 很小或总体很小时，应该用超几何分布而非二项分布。这里的“独立同分布”假设来自“大总体、小样本”。 
import numpy as np
import matplotlib.pyplot as plt
from scipy.stats import binom

np.random.seed(2)
n, p = 10, 0.60
n_sim = 10000

# 模拟：每次抽 10 颗（有放回近似），记录蓝球数
blue_counts = []

    blue_counts.append(np.sum(drawn == 'blue'))

theory = binom.pmf(k_vals, n, p)

fig, ax = plt.subplots(figsize=(8, 4))
ax.hist(blue_counts, bins=np.arange(-0.5, n + 1.5, 1),
        density=True, alpha=0.6, color='skyblue', label='Simulated')
ax.bar(k_vals + 0.2, theory, width=0.35, color='red', alpha=0.5, label='Binomial PMF')
ax.set_xlabel('Number of blue marbles in 10 draws')
ax.set_ylabel('Probability')
ax.set_title(f'Simulated vs theoretical Binomial(n={n}, p={p})')
ax.legend(); plt.tight_layout(); plt.show()

假设与探究

1. 抽样方式：“有放回”让每次抽取独立，适合用二项分布；“不放回”在总体有限时更准确，但计算复杂。你的场景更接近哪一种？

2. 样本量：大数定律保证 n 越大越准确，但成本也越高。如何在精度与计算量之间权衡？

3. 总体比例未知：真实问题中总体比例常常未知，我们只能用样本比例估计它，并给出置信区间。

2. Monte Carlo：产品上线风险（PERT / 关键路径）

一个科技初创公司要发布产品。8 项任务有依赖关系，每项任务持续时间是随机变量。CEO 想知道：平均工期、30 天内上线概率、哪些任务最可能拖后腿。

Block 1：定义任务、分布与依赖

状态与随机变量：每项任务的持续时间 d_A … d_H 是随机变量；任务的开始时间取决于其所有前置任务的最大完成时间。

规则：

  • 分布类型：Normal、Uniform、Constant
  • start_i = max{finish_j | j ∈ predecessors(i)}
  • finish_i = start_i + duration_i
  • 总工期 T = finish_H

关键假设：任务持续时间彼此独立；前置关系固定不变；资源无限（可同时并行）。

import numpy as np
import pandas as pd

np.random.seed(3)

# 任务：id, 名称, 前置任务, 持续时间的分布参数
tasks_data = [
    {"id":"A", "name":"Feature Development",   "predecessors":[],       "dist":{"type":"normal",  "mean":10, "std_dev":2}},
    {"id":"B", "name":"UI/UX Design",          "predecessors":[],       "dist":{"type":"normal",  "mean":7,  "std_dev":1.5}},
    {"id":"C", "name":"Backend Integration",   "predecessors":["A"],   "dist":{"type":"normal",  "mean":8,  "std_dev":2}},
    {"id":"D", "name":"Frontend Development",  "predecessors":["B"],   "dist":{"type":"normal",  "mean":6,  "std_dev":1}},
    {"id":"E", "name":"Testing & Bug Fixing",  "predecessors":["C","D"],"dist":{"type":"uniform", "min":5,  "max":10}},
    {"id":"F", "name":"Marketing Campaign",    "predecessors":[],       "dist":{"type":"normal",  "mean":12, "std_dev":3}},
    {"id":"G", "name":"Legal Review",          "predecessors":["A","B"],"dist":{"type":"uniform", "min":3,  "max":7}},
    {"id":"H", "name":"Product Launch (Final)","predecessors":["E","F","G"],"dist":{"type":"constant","value":1}},
]

def sample_duration(d):
    t = d["type"]
    if t == "normal":   return max(0.1, np.random.normal(d["mean"], d["std_dev"]))
    if t == "uniform":  return np.random.uniform(d["min"], d["max"])
    if t == "constant": return d["value"]

def simulate_project(tasks):
    """单次运行：返回总工期、各任务完成时间、关键路径任务集合。"""
    tasks_dict = {t["id"]: t for t in tasks}
    finish, sampled = {}, {}
    pending = list(tasks)
    processed = set()
    while pending:
        ready = [t for t in pending if all(p in processed for p in t["predecessors"])]
        pending = [t for t in pending if t not in ready]
        for t in ready:
            dur = sample_duration(t["dist"])
            sampled[t["id"]] = dur
            start = max((finish[p] for p in t["predecessors"]), default=0)
            finish[t["id"]] = start + dur
            processed.add(t["id"])
    total = finish['H']
    # 反向追踪关键路径
    critical = set(['H'])
    to_trace = ['H']
    tol = 0.001
    while to_trace:
        tid = to_trace.pop()
        start = finish[tid] - sampled[tid]
        for p in tasks_dict[tid]["predecessors"]:
            if abs(finish[p] - start) < tol:
                critical.add(p)
                to_trace.append(p)
    return total, finish, critical

print("单次示例运行：")
dur, fin, cp = simulate_project(tasks_data)
print(f"总工期 = {dur:.1f} 天，关键路径任务 = {sorted(cp)}")

Block 2：Monte Carlo 跑 10 000 次

输出指标：总工期分布、平均工期、30 天内完成概率、累积平均收敛曲线。

探究：如果只给出“平均工期”，CEO 会低估风险。分布尾部告诉我们“最差情况下可能多久”。

import numpy as np

fig, axes = plt.subplots(1, 2, figsize=(14, 4))
axes[0].hist(durations, bins=50, color='teal', alpha=0.7, edgecolor='white')
axes[0].axvline(30, color='red', ls='--', label='30-day deadline')

Block 3：关键路径概率与敏感性

输出指标：每项任务出现在关键路径上的频率；改变某任务均值后总工期的变化。

探究：“最关键”的任务不一定是耗时最长的，而是最经常决定总工期的。把资源投入到这些任务上才能最有效降低风险。

import numpy as np
import matplotlib.pyplot as plt

np.random.seed(5)
N = 10000
critical_counts = {t["id"]: 0 for t in tasks_data}
for _ in range(N):
    _, _, cp = simulate_project(tasks_data)
    for tid in cp:
        critical_counts[tid] += 1

probs = {k: v / N * 100 for k, v in critical_counts.items()}
print("关键路径概率：")
for k, v in sorted(probs.items(), key=lambda x: -x[1]):
    print(f"  {k}: {v:.1f}%")

# 敏感性：把任务 A 的 mean 分别 -10% 和 +10%，看平均工期变化
base_mean = tasks_data[0]["dist"]["mean"]
sens = {}
for delta in [-0.1, 0, 0.1]:
    tasks_data[0]["dist"]["mean"] = base_mean * (1 + delta)
    durs = [simulate_project(tasks_data)[0] for _ in range(2000)]
    sens[f"A mean {delta:+.0%}"] = np.mean(durs)
tasks_data[0]["dist"]["mean"] = base_mean  # restore

fig, axes = plt.subplots(1, 2, figsize=(14, 4))
axes[0].barh(list(probs), list(probs.values()), color='slateblue')
axes[0].set_xlabel('Probability of being on critical path (%)')
axes[0].set_title('Task criticality from Monte Carlo')
axes[1].bar(sens.keys(), sens.values(), color='coral')
axes[1].set_ylabel('Average project duration (days)')
axes[1].set_title('Sensitivity to task A mean duration')

假设与探究

1. 分布选择：为什么用 Normal 而不是 Uniform？Normal 更适合“经验均值 ± 波动”；Uniform 适合“只知道上下界”。如果任务时间有尾部风险，可考虑 log-normal。

2. 任务独立性：现实中某些任务会同时受同一外部因素影响（如团队士气）。若引入相关性，总工期方差会如何变化？

3. 资源约束：当前模型假设任务只要前置完成就能开始。若同一团队不能并行做多件事，关键路径可能完全不同。

3. Agent-Based Simulation：单车道交通流

从“每辆车的简单规则”出发，观察道路上拥堵如何自发形成。这里把车辆当作 agent，道路当作环境，规则包括：匀速前进、保持安全距离、入口按 Poisson 到达。

Block 1：最简模型——匀速前进并驶出

状态与随机变量：每辆车的位置 x；初始时刻道路上没有车。

规则：

  • 每辆车以恒定速度 v 前进
  • 若位置超过道路长度 L，则离开系统

输出指标：任意时刻道路上的车辆数、驶出车辆数。

假设：没有新车进入，所有车速度相同，不超车。

import numpy as np
import matplotlib.pyplot as plt

np.random.seed(6)
L = 1000          # 路段长度 (m)
v = 20            # 车速 (m/s)
dt = 1            # 时间步长 (s)
steps = 60

# 初始化：路上有 10 辆车，均匀分布
cars = [{'x': i * 80, 'v': v} for i in range(10)]
n_on_road = []

for t in range(steps):
    # 更新位置
    for c in cars:
        c['x'] += c['v'] * dt
    # 移除驶出车辆
    cars = [c for c in cars if c['x'] < L]
    n_on_road.append(len(cars))

plt.figure(figsize=(8, 3))
plt.plot(n_on_road, marker='o')
plt.xlabel('Time step (s)')
plt.ylabel('Cars on segment')
plt.title('Basic traffic: cars exit at constant speed')
plt.grid(True, alpha=0.3)
plt.tight_layout(); plt.show()

Block 2：加入 Poisson 到达与安全跟车

复杂化：入口处按 Poisson(λ) 产生新车；每辆车根据前车距离调整速度，保持安全车距。

新增规则：

  • 新车在 x=0 处生成，但需保证与入口最近车辆的距离 ≥ 安全车距
  • 每步速度 = min(期望速度, 到前车距离 - 安全车距)
  • 安全车距 = 车长 + 最小间隙

输出指标：平均通行时间、最大排队长度、吞吐量。

import numpy as np
import matplotlib.pyplot as plt

np.random.seed(7)
L, v, car_len, min_gap = 1000, 20, 5, 10
safe_dist = car_len + min_gap
dt, steps = 1, 300
lam = 0.4         # 每秒平均到达车辆数

cars = []         # 每辆车：{'x':位置, 'entry_t':进入时间}
travel_times = []
queue = 0         # 入口排队计数（简化为到达但被阻塞的车辆数）
max_queue = 0
n_on_road = []

for t in range(steps):
    # 1) 按 Poisson 到达新车
    arrivals = np.random.poisson(lam)
    blocked = 0
    for _ in range(arrivals):
        if not cars or cars[-1]['x'] >= safe_dist:
            cars.append({'x': 0, 'entry_t': t})
        else:
            blocked += 1
    queue += blocked
    max_queue = max(max_queue, queue)

    # 2) 从尾到头更新速度（避免碰撞）
    # 按位置从大到小排序，先更新前车
    cars.sort(key=lambda c: c['x'], reverse=True)
    for i, c in enumerate(cars):
        leader = cars[i-1] if i > 0 else None
        if leader is None:
            target_speed = v
        else:
            gap = leader['x'] - c['x'] - car_len
            target_speed = min(v, max(0, gap - min_gap) / dt)
        c['x'] += target_speed * dt

    # 3) 移除驶出车辆并记录通行时间
    cars_new = []
    for c in cars:
        if c['x'] >= L:
            travel_times.append(t - c['entry_t'])
        else:
            cars_new.append(c)
    cars = cars_new
    n_on_road.append(len(cars))

print(f"平均通行时间：{np.mean(travel_times):.1f} s")
print(f"最大排队车辆数（入口阻塞）：{max_queue}")
print(f"总吞吐量：{len(travel_times)} 辆")

fig, ax = plt.subplots(figsize=(8, 3))
ax.plot(n_on_road, color='steelblue')
ax.set_xlabel('Time step (s)')
ax.set_ylabel('Cars on segment')
ax.set_title('Traffic with Poisson arrivals and car-following')
plt.tight_layout(); plt.show()

Block 3：参数扫描——到达率如何影响拥堵

探究：固定道路长度和车速，只改变到达率 λ，观察平均通行时间和吞吐量。当 λ 超过道路容量时，会出现相变：车辆开始堆积，通行时间激增。

输出指标：不同 λ 下的平均通行时间、总吞吐量、最大排队长度。

import numpy as np
import matplotlib.pyplot as plt

def run_traffic(lam, seed=8):
    np.random.seed(seed)
    L, v, car_len, min_gap = 1000, 20, 5, 10
    safe_dist = car_len + min_gap
    dt, steps = 1, 300
    cars, travel_times, max_queue = [], [], 0
    for t in range(steps):
        arrivals = np.random.poisson(lam)
        blocked = 0
        for _ in range(arrivals):
            if not cars or cars[-1]['x'] >= safe_dist:
                cars.append({'x': 0, 'entry_t': t})
            else:
                blocked += 1
        max_queue = max(max_queue, blocked)
        cars.sort(key=lambda c: c['x'], reverse=True)
        for i, c in enumerate(cars):
            leader = cars[i-1] if i > 0 else None
            gap = (leader['x'] - c['x'] - car_len) if leader else 1e9
            target = min(v, max(0, gap - min_gap) / dt)
            c['x'] += target * dt
        cars_new = []
        for c in cars:
            if c['x'] >= L:
                travel_times.append(t - c['entry_t'])
            else:
                cars_new.append(c)
        cars = cars_new
    avg_travel = np.mean(travel_times) if travel_times else np.nan
    return avg_travel, len(travel_times), max_queue

lambdas = np.linspace(0.1, 0.8, 15)
results = [run_traffic(l) for l in lambdas]
avg_times = [r[0] for r in results]
throughputs = [r[1] for r in results]
queues = [r[2] for r in results]

fig, axes = plt.subplots(1, 3, figsize=(15, 3.5))
axes[0].plot(lambdas, avg_times, marker='o'); axes[0].set_title('Avg travel time')
axes[0].set_xlabel('Arrival rate λ'); axes[0].set_ylabel('Seconds')
axes[1].plot(lambdas, throughputs, marker='s', color='green'); axes[1].set_title('Throughput')
axes[1].set_xlabel('Arrival rate λ'); axes[1].set_ylabel('Cars exited')
axes[2].plot(lambdas, queues, marker='^', color='red'); axes[2].set_title('Max queue')
axes[2].set_xlabel('Arrival rate λ'); axes[2].set_ylabel('Cars')
plt.tight_layout(); plt.show()

假设与探究

1. 同质车辆：所有车速度和长度相同。引入快车道/慢车道或不同车型，会形成什么样的涌现行为？

4. Agent-Based Simulation：谣言 / 八卦传播

一个小社区里有若干居民。每个人每天决定“宅家”或“去公共场所”；在公共场所遇到人时，有一定概率交换彼此知道的信息。微观上的“见面聊聊”如何导致宏观上的“谣言爆发”？

Block 1：单个谣言的单次传播

状态与随机变量：每个 agent 的位置（家或公共场所）、已知的谣言集合。

规则：

  • 每个人以概率 stay_home 决定留在家中
  • 出门者随机选择一个公共场所
  • 同一场所的人 pairwise 以概率 meet_prob 交谈
  • 交谈时以 spread_prob × receive_prob 的概率交换未知谣言

输出指标：知道谣言的人数随时间变化。

import numpy as np
import matplotlib.pyplot as plt
import random

np.random.seed(9); random.seed(9)

num_agents = 30
num_places = 4
grid_size = 10
steps = 60
meet_prob = 0.7
spread_prob = 0.8
receive_prob = 0.9

class Agent:
    def __init__(self, aid):
        self.id = aid
        self.home = tuple(np.random.randint(0, grid_size, 2))
        self.loc = self.home
        self.gossip = set()
        self.stay_home = random.uniform(0.3, 0.7)

agents = [Agent(i) for i in range(num_agents)]
public_places = [tuple(np.random.randint(0, grid_size, 2)) for _ in range(num_places)]

# 初始只有 1 人知道谣言
source = random.choice(agents)
source.gossip.add(1)

history = []
for t in range(steps):
    for a in agents:
        a.loc = a.home if random.random() < a.stay_home else random.choice(public_places)
    # 按位置分组
    locs = {}
    for a in agents:
        locs.setdefault(a.loc, []).append(a)
    for group in locs.values():
        if len(group) < 2: continue
        for i in range(len(group)):
            for j in range(i+1, len(group)):
                if random.random() < meet_prob:
                    a1, a2 = group[i], group[j]
                    for g in a1.gossip:
                        if g not in a2.gossip and random.random() < spread_prob * receive_prob:
                            a2.gossip.add(g)
                    for g in a2.gossip:
                        if g not in a1.gossip and random.random() < spread_prob * receive_prob:
                            a1.gossip.add(g)
    history.append(sum(1 for a in agents if 1 in a.gossip))

plt.figure(figsize=(8, 4))
plt.plot(range(1, steps+1), history, marker='o', color='purple')
plt.xlabel('Time step'); plt.ylabel('Agents knowing gossip')
plt.title('Gossip spread in a small community')
plt.grid(True, alpha=0.3)

探究：改变“见面即交谈”的概率 meet_prob，观察谣言达到 80% 人群所需时间。即使 meet_prob 不高，只要时间足够，谣言仍可能传遍社区。

5. Agent-Based Simulation：森林大火

森林大火是经典的元胞自动机 / ABS 模型。树木是环境细胞，火是“活性 agent”。从最简单的确定性蔓延开始，逐步加入概率、再生和长期演化。

Block 1：确定性火源扩散

状态：EMPTY=0, TREE=1, BURNING=2, ASH=3。

规则：

  • BURNING → ASH
  • TREE 且 8 邻域有 BURNING → BURNING

输出指标：若干时间步后的空间格局。

import numpy as np
import matplotlib.pyplot as plt
from matplotlib.colors import ListedColormap

np.random.seed(11)
EMPTY, TREE, BURNING, ASH = 0, 1, 2, 3
CMAP = ListedColormap(['white', 'green', 'red', 'black'])

def init_forest(size, density):
    f = np.zeros((size, size), dtype=int)
    f[np.random.rand(size, size) < density] = TREE
    f[size//2, size//2] = BURNING
    return f

def step_det(f):
    new = f.copy()
    burn = (f == BURNING)
    new[burn] = ASH
    for dr in (-1, 0, 1):
        for dc in (-1, 0, 1):
            if dr == 0 and dc == 0: continue
            new[(f == TREE) & np.roll(burn, (dr, dc), axis=(0, 1))] = BURNING
    return new

size, density, steps = 21, 0.6, 10
f = init_forest(size, density)
frames = [f.copy()]
for _ in range(steps):
    f = step_det(f); frames.append(f.copy())

fig, axes = plt.subplots(1, 4, figsize=(14, 3.5))
for ax, idx in zip(axes, [0, 3, 6, 10]):
    ax.imshow(frames[idx], cmap=CMAP, vmin=0, vmax=3)
    ax.set_title(f'Step {idx}'); ax.axis('off')
plt.suptitle('Deterministic fire spread')
plt.tight_layout(); plt.show()

Block 2：概率蔓延与随机起火

复杂化：引入 p_spread（被邻居点燃的概率）和 p_ignite（每棵树自发燃烧的概率）。

输出指标：不同初始树密度下的平均烧毁比例。

import numpy as np
import matplotlib.pyplot as plt

np.random.seed(12)
EMPTY, TREE, BURNING, ASH = 0, 1, 2, 3

def init_f(size, d, p_ignite):
    f = (np.random.rand(size, size) < d).astype(int)
    ignite = np.random.rand(size, size) < p_ignite * 10
    f[(f == TREE) & ignite] = BURNING
    return f

def step_prob(f, p_spread, p_ignite):
    new = f.copy()
    burn = (f == BURNING)
    new[burn] = ASH
    risk = np.zeros_like(f, dtype=float)
    for dr in (-1, 0, 1):
        for dc in (-1, 0, 1):
            if dr == 0 and dc == 0: continue
            risk += np.roll(burn, (dr, dc), axis=(0, 1)).astype(float)
    catch = np.random.rand(*f.shape) < 1 - (1 - p_spread)**risk
    new[(f == TREE) & catch] = BURNING
    new[(f == TREE) & (np.random.rand(*f.shape) < p_ignite)] = BURNING
    return new

def burned_frac(d, size=41, steps=30, reps=10):
    vals = []
    for _ in range(reps):
        f = init_f(size, d, 0.001)
        for _ in range(steps): f = step_prob(f, 0.7, 0.001)
        vals.append((f == ASH).sum() / f.size)
    return np.mean(vals)

densities = np.arange(0.1, 0.9, 0.03)
ys = [burned_frac(d) for d in densities]

plt.figure(figsize=(8, 4))
plt.plot(densities, ys, marker='o', color='firebrick')
plt.xlabel('Initial tree density'); plt.ylabel('Average burned fraction')
plt.title('Burned fraction vs. tree density')
plt.grid(True, ls='--', alpha=0.5)
plt.tight_layout(); plt.show()

6. Agent-Based Simulation：Schelling 隔离模型

即使每个人只要求“同类邻居占一定比例”，整个社区也会自发走向高度隔离。Schelling 模型揭示了“微观偏好 → 宏观隔离”的涌现机制。

Block 1：基础两类型隔离

状态：每个格子为空或属于 A/B 两类居民。

规则：

  • 每个居民检查 8 邻域中同类占非空邻居的比例
  • 若比例低于容忍阈值 T，则随机搬到空房
  • 重复直到无人搬家或达到最大迭代次数

输出指标：平均相似度 = 所有居民同类邻居比例的均值。

import numpy as np
import matplotlib.pyplot as plt
import random

np.random.seed(14); random.seed(14)
EMPTY = 0

def init_grid(size, empty_ratio, ratios):
    total = size * size
    n_empty = int(total * empty_ratio)
    counts = [int(total * r) for r in ratios]
    diff = total - n_empty - sum(counts)
    if diff > 0: n_empty += diff
    elif diff < 0: counts[-1] += diff
    cells = [EMPTY]*n_empty
    for t, n in enumerate(counts, start=1):
        cells += [t]*n
    random.shuffle(cells)
    return np.array(cells).reshape(size, size)

def similarity(grid):
    size = grid.shape[0]
    total, n = 0.0, 0
    for r in range(size):
        for c in range(size):
            a = grid[r, c]
            if a == EMPTY: continue
            n += 1
            neighbors = []
            for dr in (-1,0,1):
                for dc in (-1,0,1):
                    if dr==0 and dc==0: continue
                    nr, nc = r+dr, c+dc
                    if 0 <= nr < size and 0 <= nc < size:
                        neighbors.append(grid[nr, nc])
            if neighbors:
                total += neighbors.count(a) / len(neighbors)
    return 100 * total / n if n else 0

def is_happy(grid, r, c, T):
    a = grid[r, c]
    if a == EMPTY: return True
    size = grid.shape[0]
    rel = [grid[nr, nc] for dr in (-1,0,1) for dc in (-1,0,1)
           if not (dr==0 and dc==0) and 0 <= (nr:=r+dr) < size and 0 <= (nc:=c+dc) < size
           and grid[nr, nc] != EMPTY]
    if not rel: return True
    return rel.count(a) / len(rel) >= T

def step(grid, T):
    size = grid.shape[0]
    unhappy = [(r,c) for r in range(size) for c in range(size)
               if grid[r,c]!=EMPTY and not is_happy(grid,r,c,T)]
    empty = [(r,c) for r in range(size) for c in range(size) if grid[r,c]==EMPTY]
    random.shuffle(unhappy); random.shuffle(empty)
    for r,c in unhappy:
        if not empty: break
        nr, nc = empty.pop()
        empty.append((r,c))
        grid[nr,nc] = grid[r,c]
        grid[r,c] = EMPTY
    return grid

size, er, T = 40, 0.2, 0.3
grid = init_grid(size, er, [0.4, 0.4])
print(f"Initial similarity: {similarity(grid):.1f}%")
fig, axes = plt.subplots(1, 2, figsize=(12, 5))
axes[0].imshow(grid, cmap='viridis', vmin=0, vmax=2)
axes[0].set_title('Initial'); axes[0].axis('off')

for _ in range(50):
    grid = step(grid, T)

axes[1].imshow(grid, cmap='viridis', vmin=0, vmax=2)
axes[1].set_title(f'Final (T={T})'); axes[1].axis('off')
print(f"Final similarity: {similarity(grid):.1f}%")
plt.tight_layout(); plt.show()

7. 概率优化：模拟退火（Simulated Annealing）

模拟退火是一种“带随机扰动的贪心搜索”：以一定概率接受更差的解，从而跳出局部最优。它常用于组合优化，如选址、调度、路径规划。

Block 1：公交站电气化选址

问题：有 30 个公交场站，每个场站电气化需要成本和带来收益。总预算有限，选择哪些场站才能最大化总收益？

模拟退火要素：

  • 状态：长度为 30 的二进制向量，1 表示选中
  • 目标函数：选中场站收益之和（若超预算则 -∞）
  • 邻域：随机翻转一个比特，或交换选中/未选中
  • 温度 T：高温时容易接受差解，低温时趋于贪心
  • 接受概率：P = exp((new_obj - old_obj) / T)

输出指标：最优解、最优目标值、收敛曲线。

import numpy as np
import random
import math
import matplotlib.pyplot as plt

np.random.seed(17); random.seed(17)

num_depots = 30
budget = 1500
costs = [random.randint(50, 150) for _ in range(num_depots)]
benefits = [random.randint(30, 80) for _ in range(num_depots)]

def objective(state):
    c = sum(costs[i] for i, s in enumerate(state) if s)
    b = sum(benefits[i] for i, s in enumerate(state) if s)
    return -float('inf') if c > budget else b

def neighbor(state):
    s = list(state)
    if random.random() < 0.7:
        s[random.randint(0, num_depots-1)] ^= 1
    else:
        on = [i for i, v in enumerate(s) if v]
        off = [i for i, v in enumerate(s) if not v]
        if on and off:
            s[random.choice(on)] = 0
            s[random.choice(off)] = 1
    return tuple(s)

# 初始解：随机选约一半，若超预算则随机移除
state = [0]*num_depots
for _ in range(num_depots//2):
    state[random.randint(0, num_depots-1)] = 1
while sum(costs[i] for i, v in enumerate(state) if v) > budget and any(state):
    on = [i for i, v in enumerate(state) if v]
    state[random.choice(on)] = 0
state = tuple(state)

T = 5000
cooling = 0.999
best_state, best_obj = state, objective(state)
current_state, current_obj = state, best_obj
hist_current, hist_best = [], []

for _ in range(20000):
    nb = neighbor(current_state)
    nb_obj = objective(nb)
    if nb_obj > current_obj:
        accept_prob = 1.0
    elif T <= 0:
        accept_prob = 0.0
    else:
        accept_prob = math.exp((nb_obj - current_obj) / T)
    if random.random() < accept_prob:
        current_state, current_obj = nb, nb_obj
    if current_obj > best_obj:
        best_state, best_obj = current_state, current_obj
    T *= cooling
    hist_current.append(current_obj)
    hist_best.append(best_obj)
    if T < 1: break

print(f"Best benefit = {best_obj}, cost = {sum(costs[i] for i,v in enumerate(best_state) if v)}")

plt.figure(figsize=(8, 4))
plt.plot(hist_current, alpha=0.5, label='Current objective')
plt.plot(hist_best, label='Best objective')
plt.xlabel('Iteration'); plt.ylabel('Total benefit')
plt.title('Simulated annealing convergence')
plt.legend(); plt.tight_layout(); plt.show()

Block 2：冷却速度对收敛的影响

探究：冷却率 cooling_rate 决定温度下降快慢。冷却太快容易陷入局部最优；冷却太慢计算成本高。比较 0.99、0.995、0.999 三种冷却速度。

import numpy as np
import random
import math
import matplotlib.pyplot as plt

np.random.seed(18); random.seed(18)

def run_sa(cooling, max_iter=20000):
    T = 5000
    state = tuple([1 if random.random() < 0.5 else 0 for _ in range(num_depots)])
    while sum(costs[i] for i, v in enumerate(state) if v) > budget and any(state):
        on = [i for i, v in enumerate(state) if v]
        lst = list(state); lst[random.choice(on)] = 0; state = tuple(lst)
    cur, cur_obj = state, objective(state)
    best, best_obj = cur, cur_obj
    hist = [best_obj]
    for _ in range(max_iter):
        nb = neighbor(cur)
        nb_obj = objective(nb)
        if nb_obj > cur_obj or (T > 0 and random.random() < math.exp((nb_obj - cur_obj)/T)):
            cur, cur_obj = nb, nb_obj
        if cur_obj > best_obj:
            best, best_obj = cur, cur_obj
        T *= cooling
        hist.append(best_obj)
        if T < 1: break
    return hist

plt.figure(figsize=(8, 4))
for cr in [0.99, 0.995, 0.999]:
    hist = run_sa(cr)
    plt.plot(hist, label=f'cooling={cr}')
plt.xlabel('Iteration'); plt.ylabel('Best benefit')
plt.title('Impact of cooling rate on SA convergence')
plt.legend(); plt.tight_layout(); plt.show()

假设与探究

1. 邻域设计：只翻转单个比特 vs 交换选中/未选中，哪种更容易保持预算可行性？

2. 初始温度：温度过高会浪费迭代，过低则失去跳出局部最优的能力。如何根据目标函数量级选择 T₀？

3. 最优性：模拟退火不保证全局最优。对小型问题可对比动态规划或穷举，验证解的质量。

8. 仿真模型选择与建模起手式

方法 核心思想 适用场景 关键输出
Monte Carlo 重复随机抽样，用频率逼近概率 风险分析、工期、面积估计、比例推断 分布、概率、置信区间、敏感性
