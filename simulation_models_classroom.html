0. 仿真模型是什么？一个统一的四步框架

确定性模型对同样的输入永远给出同样的输出；仿真模型则把不确定性直接写进模型里，回答的是“结果会怎么分布、风险有多大”。

无论 Monte Carlo、Agent-Based 还是概率优化，构造一个仿真模型都可以归为四步：

  1. 状态与随机变量：模型追踪什么？哪些是随机抽取的？
  2. 规则 / 分布：变量服从什么分布？agent 按什么规则更新？
  3. 输出指标：我们想估计的数值是什么？
  4. 参数与假设：哪些数是暂时固定的？改变它们会产生什么洞察？
探究假设的过程

仿真的价值不在于“跑一次”，而在于改变参数再跑。每次改动一个假设（例如把到达率 λ 变大、把传播概率降低、把冷却速度加快），观察输出指标如何移动，就能判断哪些假设对结论最敏感。

takeaway：好的仿真模型 = 清晰的假设 + 可重复的代码 + 对参数的系统性扫描。 
import numpy as np
import matplotlib.pyplot as plt

# --- Parameters (user-tunable) ---
params = {
    'mu': 5.0,      # mean of the random input distribution
    'sigma': 2.0,   # standard deviation of the random input distribution
}
N_RUNS = 10000     # number of Monte Carlo replications

# --- Generic Monte Carlo skeleton ---
def simulate_once(params):
    """单次仿真：根据参数生成随机输入并计算输出。"""
    x = np.random.normal(params['mu'], params['sigma'])
    return max(0, x)            # 例如：风速不能为负

def run_simulation(params, n_runs=N_RUNS):
    """重复 n_runs 次，返回输出分布。"""
    results = [simulate_once(params) for _ in range(n_runs)]
    return np.array(results)

out = run_simulation(params)
print(f"Mean={out.mean():.2f}, Std={out.std():.2f}, P>7={np.mean(out>7):.2%}")

1. Monte Carlo：弹珠抽样与大数定律

袋子里有 10% 红、30% 黄、60% 蓝弹珠。我们无法一次性清点整袋，于是不断随机抽取，用样本比例逼近真实比例。这是 Monte Carlo 最直观的形态。

Block 1：构建总体并单次抽样

状态与随机变量：每颗弹珠的颜色是离散的随机变量；总体比例已知，样本比例未知。

模型规则：

  • 总体：red=0.10, yellow=0.30, blue=0.60
  • 每次从袋中不放回地抽取 n 颗
  • 计算样本中各颜色占比

假设：袋子足够大（100 000 颗），单次抽样的 n 远小于总体，因此可近似看作独立同分布抽样。

import numpy as np
from collections import Counter

# --- Parameters (user-tunable) ---
TOTAL_MARBLES = 100000   # total marbles in the bag
RED_PCT = 0.10           # true proportion of red marbles
YELLOW_PCT = 0.30        # true proportion of yellow marbles
BLUE_PCT = 0.60          # true proportion of blue marbles
DRAW_SMALL = 30          # small sample size for demonstration
DRAW_LARGE = 5000        # large sample size for demonstration

np.random.seed(0)
# 构建总体
marbles = (
    ['red'] * int(TOTAL_MARBLES * RED_PCT) +
    ['yellow'] * int(TOTAL_MARBLES * YELLOW_PCT) +
    ['blue'] * int(TOTAL_MARBLES * BLUE_PCT)
)
np.random.shuffle(marbles)

# Function: draw_and_report - draw n marbles and report sample color proportions
def draw_and_report(n):
    drawn = np.random.choice(marbles, size=n, replace=False)
    counts = Counter(drawn)
    # Loop: iterate over [('red',10),('yellow',30),('blue',60)]
    for color, true_p in [('red', 10), ('yellow', 30), ('blue', 60)]:
        pct = counts.get(color, 0) / n * 100
        print(f"{color:6}: {counts.get(color, 0):4d} ({pct:5.2f}%)  true={true_p}%")

print("--- Draw n=30 ---")
draw_and_report(DRAW_SMALL)
print("\n--- Draw n=5000 ---")
draw_and_report(DRAW_LARGE)

Block 3：用二项分布刻画“抽到蓝球”

模型规则：每次只关心“是否抽到蓝球”，则 n 次独立抽取中蓝球数量 X ~ Binomial(n, p=0.6)。

输出指标：模拟频率与理论概率质量函数 PMF 的对比。

假设再探：当 n 很小或总体很小时，应该用超几何分布而非二项分布。这里的“独立同分布”假设来自“大总体、小样本”。

import matplotlib.pyplot as plt

P_BLUE = 0.60           # true probability of drawing a blue marble
N_SIMULATIONS = 10000   # number of repeated experiments

np.random.seed(2)

    ['yellow'] * int(TOTAL_MARBLES * 0.30) +
    ['blue'] * int(TOTAL_MARBLES * P_BLUE)

np.random.shuffle(marbles)

# 模拟：每次抽 N_TRIALS 颗，记录蓝球数
blue_counts = []
# Loop: iterate range(N_SIMULATIONS) times

theory = binom.pmf(k_vals, N_TRIALS, P_BLUE)

ax.bar(k_vals + 0.2, theory, width=0.35, color='red', alpha=0.5, label='Binomial PMF')
ax.set_xlabel('Number of blue marbles in N_TRIALS draws')
ax.set_ylabel('Probability')
ax.set_title(f'Simulated vs theoretical Binomial(n={N_TRIALS}, p={P_BLUE})')
ax.legend(); plt.tight_layout(); plt.show()

假设与探究

1. 抽样方式：“有放回”让每次抽取独立，适合用二项分布；“不放回”在总体有限时更准确，但计算复杂。你的场景更接近哪一种？

2. 样本量：大数定律保证 n 越大越准确，但成本也越高。如何在精度与计算量之间权衡？

3. 总体比例未知：真实问题中总体比例常常未知，我们只能用样本比例估计它，并给出置信区间。

2. Monte Carlo：产品上线风险（PERT / 关键路径）

一个科技初创公司要发布产品。8 项任务有依赖关系，每项任务持续时间是随机变量。CEO 想知道：平均工期、30 天内上线概率、哪些任务最可能拖后腿。

任务依赖网络示意（箭头表示前置关系）

Block 1：定义任务、分布与依赖

状态与随机变量：每项任务的持续时间 d_A … d_H 是随机变量；任务的开始时间取决于其所有前置任务的最大完成时间。

规则：

  • 分布类型：Normal、Uniform、Constant
  • start_i = max{finish_j | j ∈ predecessors(i)}
  • finish_i = start_i + duration_i
  • 总工期 T = finish_H

关键假设：任务持续时间彼此独立；前置关系固定不变；资源无限（可同时并行）。

import numpy as np

# --- Parameters (user-tunable) ---
# Task definitions: id, name, predecessors, duration distribution
TASKS_DATA = [
    {"id":"A", "name":"Feature Development",   "predecessors":[],       "dist":{"type":"normal",  "mean":10, "std_dev":2}},
    {"id":"B", "name":"UI/UX Design",          "predecessors":[],       "dist":{"type":"normal",  "mean":7,  "std_dev":1.5}},
    {"id":"C", "name":"Backend Integration",   "predecessors":["A"],   "dist":{"type":"normal",  "mean":8,  "std_dev":2}},
    {"id":"D", "name":"Frontend Development",  "predecessors":["B"],   "dist":{"type":"normal",  "mean":6,  "std_dev":1}},
    {"id":"E", "name":"Testing & Bug Fixing",  "predecessors":["C","D"],"dist":{"type":"uniform", "min":5,  "max":10}},
    {"id":"F", "name":"Marketing Campaign",    "predecessors":[],       "dist":{"type":"normal",  "mean":12, "std_dev":3}},
    {"id":"G", "name":"Legal Review",          "predecessors":["A","B"],"dist":{"type":"uniform", "min":3,  "max":7}},
    {"id":"H", "name":"Product Launch (Final)","predecessors":["E","F","G"],"dist":{"type":"constant","value":1}},
]

# Function: sample_duration - draw a random duration for a task from its chosen distribution
def sample_duration(d):
    t = d["type"]
    if t == "normal":   return max(0.1, np.random.normal(d["mean"], d["std_dev"]))
    if t == "uniform":  return np.random.uniform(d["min"], d["max"])
    if t == "constant": return d["value"]

def simulate_project(tasks):
    """单次运行：返回总工期、各任务完成时间、关键路径任务集合。"""
    tasks_dict = {t["id"]: t for t in tasks}
    finish, sampled = {}, {}
    pending = list(tasks)
    processed = set()
    while pending:
        ready = [t for t in pending if all(p in processed for p in t["predecessors"])]
        pending = [t for t in pending if t not in ready]
        for t in ready:
            dur = sample_duration(t["dist"])
            sampled[t["id"]] = dur
            start = max((finish[p] for p in t["predecessors"]), default=0)
            finish[t["id"]] = start + dur
            processed.add(t["id"])
    total = finish['H']
    # 反向追踪关键路径
    critical = set(['H'])
    to_trace = ['H']
    tol = 0.001
    while to_trace:
        tid = to_trace.pop()
        start = finish[tid] - sampled[tid]
        for p in tasks_dict[tid]["predecessors"]:
            if abs(finish[p] - start) < tol:
                critical.add(p)
                to_trace.append(p)
    return total, finish, critical

print("单次示例运行：")
dur, fin, cp = simulate_project(TASKS_DATA)
print(f"总工期 = {dur:.1f} 天，关键路径任务 = {sorted(cp)}")

假设与探究

1. 分布选择：为什么用 Normal 而不是 Uniform？Normal 更适合“经验均值 ± 波动”；Uniform 适合“只知道上下界”。如果任务时间有尾部风险，可考虑 log-normal。

2. 任务独立性：现实中某些任务会同时受同一外部因素影响（如团队士气）。若引入相关性，总工期方差会如何变化？

3. 资源约束：当前模型假设任务只要前置完成就能开始。若同一团队不能并行做多件事，关键路径可能完全不同。

3. Agent-Based Simulation：单车道交通流

从“每辆车的简单规则”出发，观察道路上拥堵如何自发形成。这里把车辆当作 agent，道路当作环境，规则包括：匀速前进、保持安全距离、入口按 Poisson 到达。

Block 1：最简模型——匀速前进并驶出

状态与随机变量：每辆车的位置 x；初始时刻道路上没有车。

规则：

Block 2：加入 Poisson 到达与安全跟车

复杂化：入口处按 Poisson(λ) 产生新车；每辆车根据前车距离调整速度，保持安全车距。

新增规则：

  • 新车在 x=0 处生成，但需保证与入口最近车辆的距离 ≥ 安全车距
  • 每步速度 = min(期望速度, 到前车距离 - 安全车距)
  • 安全车距 = 车长 + 最小间隙

输出指标：平均通行时间、最大排队长度、吞吐量。

import matplotlib.pyplot as plt

# --- Parameters (user-tunable) ---
ROAD_LENGTH = 1000     # length of road segment (m)
CAR_SPEED = 20         # free-flow speed (m/s)
CAR_LENGTH = 5         # physical length of each car (m)
MIN_GAP = 10           # minimum gap between cars (m)
SAFE_DISTANCE = CAR_LENGTH + MIN_GAP
TIME_STEP = 1          # simulation time step (s)
N_STEPS = 300          # total simulation steps
ARRIVAL_RATE = 0.4     # Poisson arrival rate (cars per time step)

np.random.seed(7)
cars = []
travel_times = []
max_queue = 0
n_on_road = []

# Loop: iterate range(N_STEPS) times
for t in range(N_STEPS):
    # 1) 按 Poisson 到达新车
    arrivals = np.random.poisson(ARRIVAL_RATE)
    blocked = 0
    # Loop: iterate range(arrivals) times
    for _ in range(arrivals):
        if not cars or cars[-1]['x'] >= SAFE_DISTANCE:
            cars.append({'x': 0, 'entry_t': t})
        else:
            blocked += 1
    max_queue = max(max_queue, blocked)

    # 2) 从尾到头更新速度（避免碰撞）
    cars.sort(key=lambda c: c['x'], reverse=True)
    for i, c in enumerate(cars):
        leader = cars[i-1] if i > 0 else None
        if leader is None:
            target_speed = CAR_SPEED
        else:

            travel_times.append(t - c['entry_t'])
        else:
            cars_new.append(c)
    cars = cars_new
    n_on_road.append(len(cars))

Block 1：单个谣言的单次传播

import matplotlib.pyplot as plt

# --- Parameters (user-tunable) ---

P_SPREAD = 0.70          # probability fire spreads to a neighboring tree
P_IGNITE = 0.001         # spontaneous ignition probability per tree per step
N_STEPS = 30             # steps per simulation
N_REPS = 10              # repetitions per density for averaging
DENSITIES = np.arange(0.1, 0.9, 0.03)  # initial tree densities to scan

np.random.seed(12)
EMPTY, TREE, BURNING, ASH = 0, 1, 2, 3

# Function: init_f - initialize the forest with random spontaneous ignition
def init_f(size, d, p_ignite):
    f = (np.random.rand(size, size) < d).astype(int)
    ignite = np.random.rand(size, size) < p_ignite * 10
    f[(f == TREE) & ignite] = BURNING
    return f

def step_prob(f, p_spread, p_ignite):
    new = f.copy()
    burn = (f == BURNING)
    new[burn] = ASH
    risk = np.zeros_like(f, dtype=float)
    for dr in (-1, 0, 1):
        for dc in (-1, 0, 1):
            if dr == 0 and dc == 0: continue
            risk += np.roll(burn, (dr, dc), axis=(0, 1)).astype(float)
    catch = np.random.rand(*f.shape) < 1 - (1 - p_spread)**risk
    new[(f == TREE) & catch] = BURNING
    new[(f == TREE) & (np.random.rand(*f.shape) < p_ignite)] = BURNING
    return new

def burned_frac(d, size=GRID_SIZE, steps=N_STEPS, reps=N_REPS):
    vals = []
    for _ in range(reps):
        f = init_f(size, d, P_IGNITE)
        for _ in range(steps): f = step_prob(f, P_SPREAD, P_IGNITE)
        vals.append((f == ASH).sum() / f.size)
    return np.mean(vals)

6. Agent-Based Simulation：Schelling 隔离模型

即使每个人只要求“同类邻居占一定比例”，整个社区也会自发走向高度隔离。Schelling 模型揭示了“微观偏好 → 宏观隔离”的涌现机制。

Block 1：基础两类型隔离

    for r in range(size):

            a = grid[r, c]

7. 概率优化：模拟退火（Simulated Annealing）

模拟退火是一种“带随机扰动的贪心搜索”：以一定概率接受更差的解，从而跳出局部最优。它常用于组合优化，如选址、调度、路径规划。

Block 1：公交站电气化选址

问题：有 30 个公交场站，每个场站电气化需要成本和带来收益。总预算有限，选择哪些场站才能最大化总收益？

模拟退火要素：

  • 状态：长度为 30 的二进制向量，1 表示选中
  • 目标函数：选中场站收益之和（若超预算则 -∞）
  • 邻域：随机翻转一个比特，或交换选中/未选中
  • 温度 T：高温时容易接受差解，低温时趋于贪心
  • 接受概率：P = exp((new_obj - old_obj) / T)

输出指标：最优解、最优目标值、收敛曲线。

import numpy as np
import random
import math
import matplotlib.pyplot as plt

# --- Parameters (user-tunable) ---
NUM_DEPOTS = 30          # number of candidate depots
BUDGET = 1500            # total electrification budget
INITIAL_TEMP = 5000      # starting annealing temperature
COOLING_RATE = 0.999     # temperature multiplier per iteration
MAX_ITERATIONS = 20000   # total annealing iterations
STOP_TEMP = 1.0          # stop when temperature falls below this

np.random.seed(17); random.seed(17)

costs = [random.randint(50, 150) for _ in range(NUM_DEPOTS)]
benefits = [random.randint(30, 80) for _ in range(NUM_DEPOTS)]

# Function: objective - evaluate total benefit while respecting the budget constraint
def objective(state):
    c = sum(costs[i] for i, s in enumerate(state) if s)
    b = sum(benefits[i] for i, s in enumerate(state) if s)
    return -float('inf') if c > BUDGET else b

def neighbor(state):
    s = list(state)
    if random.random() < 0.7:
        s[random.randint(0, NUM_DEPOTS-1)] ^= 1
    else:
        on = [i for i, v in enumerate(s) if v]
        off = [i for i, v in enumerate(s) if not v]
        if on and off:
            s[random.choice(on)] = 0
            s[random.choice(off)] = 1
    return tuple(s)

# 初始解：随机选约一半，若超预算则随机移除
state = [0]*NUM_DEPOTS
for _ in range(NUM_DEPOTS//2):
    state[random.randint(0, NUM_DEPOTS-1)] = 1
while sum(costs[i] for i, v in enumerate(state) if v) > BUDGET and any(state):
    on = [i for i, v in enumerate(state) if v]
    state[random.choice(on)] = 0
state = tuple(state)

T = INITIAL_TEMP
best_state, best_obj = state, objective(state)
current_state, current_obj = state, best_obj
hist_current, hist_best = [], []

for _ in range(MAX_ITERATIONS):
    nb = neighbor(current_state)
    nb_obj = objective(nb)
    if nb_obj > current_obj:
        accept_prob = 1.0
    elif T <= 0:
        accept_prob = 0.0
    else:
        accept_prob = math.exp((nb_obj - current_obj) / T)
    if random.random() < accept_prob:
        current_state, current_obj = nb, nb_obj
    if current_obj > best_obj:
        best_state, best_obj = current_state, current_obj
    T *= COOLING_RATE
    hist_current.append(current_obj)
    hist_best.append(best_obj)
    if T < STOP_TEMP: break

print(f"Best benefit = {best_obj}, cost = {sum(costs[i] for i,v in enumerate(best_state) if v)}")

plt.figure(figsize=(8, 4))
plt.plot(hist_current, alpha=0.5, label='Current objective')
plt.plot(hist_best, label='Best objective')
plt.xlabel('Iteration'); plt.ylabel('Total benefit')
plt.title('Simulated annealing convergence')
plt.legend(); plt.tight_layout(); plt.show()

import numpy as np
import random
import math
import matplotlib.pyplot as plt

# --- Parameters (user-tunable) ---
COOLING_RATES = [0.99, 0.995, 0.999]  # cooling schedules to compare
MAX_ITERATIONS = 20000
INITIAL_TEMP = 5000
STOP_TEMP = 1.0
NUM_DEPOTS = 30
BUDGET = 1500

# Same random instance as Block 1 (fixed seed for comparability)
random.seed(17); np.random.seed(17)
costs = [random.randint(50, 150) for _ in range(NUM_DEPOTS)]
benefits = [random.randint(30, 80) for _ in range(NUM_DEPOTS)]

# Function: objective - evaluate total benefit while respecting the budget constraint
def objective(state):
    c = sum(costs[i] for i, s in enumerate(state) if s)
    b = sum(benefits[i] for i, s in enumerate(state) if s)
    return -float('inf') if c > BUDGET else b

def neighbor(state):
    s = list(state)
    if random.random() < 0.7:
        s[random.randint(0, NUM_DEPOTS-1)] ^= 1
    else:
        on = [i for i, v in enumerate(s) if v]
        off = [i for i, v in enumerate(s) if not v]
        if on and off:
            s[random.choice(on)] = 0
            s[random.choice(off)] = 1
    return tuple(s)

def run_sa(cooling, max_iter=MAX_ITERATIONS):
    T = INITIAL_TEMP
    state = tuple([1 if random.random() < 0.5 else 0 for _ in range(NUM_DEPOTS)])
    while sum(costs[i] for i, v in enumerate(state) if v) > BUDGET and any(state):
        on = [i for i, v in enumerate(state) if v]
        lst = list(state); lst[random.choice(on)] = 0; state = tuple(lst)
    cur, cur_obj = state, objective(state)
    best, best_obj = cur, cur_obj
    hist = [best_obj]
    for _ in range(max_iter):
        nb = neighbor(cur)
        nb_obj = objective(nb)
        if nb_obj > cur_obj or (T > 0 and random.random() < math.exp((nb_obj - cur_obj)/T)):
            cur, cur_obj = nb, nb_obj
        if cur_obj > best_obj:
            best, best_obj = cur, cur_obj
        T *= cooling
        hist.append(best_obj)
        if T < STOP_TEMP: break
    return hist
