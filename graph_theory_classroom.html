1. 什么是图？G = (V, E)

图论把现实问题抽象成点（Vertices/Nodes）和边（Edges）。点代表对象，边代表对象之间的关系。

  • V（顶点集）：图中所有节点的集合，例如建筑物、城市、课程、人群。
  • E（边集）：连接两个节点的边，可以是有向或无向，可以带权重。
  • 权重：边或点上的“代价/距离/时间/容量”。
基本定义

G = (V, E)

其中 V = {v₁, v₂, ..., vₙ}，E ⊆ V × V。

如果边带权重：G = (V, E, w)，w: E → ℝ。

takeaway：拿到题先问：谁是点？谁是边？边有没有方向？有没有权重？ 
import networkx as nx
import matplotlib.pyplot as plt

# 创建一个无向图
G = nx.Graph()
G.add_nodes_from(['A', 'B', 'C', 'D'])
G.add_weighted_edges_from([
    ('A', 'B', 4),
    ('A', 'C', 2),
    ('B', 'C', 1),
    ('B', 'D', 5),
    ('C', 'D', 8)
])

pos = nx.spring_layout(G, seed=42)
plt.figure(figsize=(6, 4))
nx.draw(G, pos, with_labels=True, node_color='skyblue', node_size=1200, font_size=12, font_weight='bold')
edge_labels = nx.get_edge_attributes(G, 'weight')
nx.draw_networkx_edge_labels(G, pos, edge_labels=edge_labels)
plt.title('Weighted Undirected Graph G = (V, E, w)')
plt.axis('off')
plt.show()

print('Vertices:', list(G.nodes))
print('Edges:', list(G.edges(data=True)))

1b. 常见图类型速查

无向图 Undirected

边没有方向，A–B 与 B–A 是同一条边。适合“连接关系”。

有向图 Directed

边有方向，A→B ≠ B→A。适合“单向流动/依赖”。

加权图 Weighted

每条边有权重（距离/时间/成本）。

完全图 Complete

任意两点之间都有边。n 个节点的完全图有 n(n−1)/2 条边。

树 Tree / 生成树 Spanning Tree

无环连通图。生成树包含原图所有顶点且边数 = n−1。

二分图 Bipartite

顶点可分为两组，边只跨组不连组内。适合“任务分配”。

2. 最短路径：Dijkstra

问题：给定加权图和起点 s，求 s 到所有其他顶点的最短距离。

数学表达

决策变量：设 d[v] 为从 s 到 v 的最短距离。

目标：最小化 d[t]（若只关心终点 t），或求出所有 d[v]。

递推关系（三角不等式）：

d[v] ≤ d[u] + w(u, v)   ∀(u, v) ∈ E

初始化：d[s] = 0，d[v] = +∞（v ≠ s）。

takeaway：Dijkstra 要求边权非负。时间复杂度 O((V+E) log V)。 
import networkx as nx

# 使用 NetworkX 内置 Dijkstra
G = nx.Graph()
G.add_weighted_edges_from([
    ('A', 'B', 4),
    ('A', 'C', 2),
    ('B', 'C', 1),
    ('B', 'D', 5),
    ('C', 'D', 8),
    ('C', 'E', 10),
    ('D', 'E', 2),
    ('D', 'F', 6),
    ('E', 'F', 3)
])

start, end = 'A', 'F'
length, path = nx.single_source_dijkstra(G, start, end)
print(f'Shortest path from {start} to {end}: {path}')
print(f'Total distance: {length}')

# 到所有点的距离
print('\nAll shortest distances from A:')
print(nx.single_source_dijkstra_path_length(G, start))

3. 最小生成树：Minimum Spanning Tree

问题：选择一组边，连接图中所有顶点，且总权重最小。典型场景：布线、道路、管道。

数学表达

决策变量：xe ∈ {0, 1}，边 e 是否被选中。

目标函数：

min Σe∈E we · xe

约束：

  • 选中 n−1 条边：Σ xe = n − 1
  • 无环（对于任意非空子集 S ⊂ V，S 内部的边数 ≤ |S| − 1）
  • 连通性：选中边构成的子图连接所有顶点
takeaway：经典算法 Kruskal（按权重贪心加边）和 Prim（从一点逐步扩展）。 
import networkx as nx
import matplotlib.pyplot as plt

G = nx.Graph()
G.add_weighted_edges_from([
    ('A', 'B', 4), ('A', 'C', 2), ('B', 'C', 1),
    ('B', 'D', 5), ('C', 'D', 8), ('C', 'E', 10),
    ('D', 'E', 2), ('D', 'F', 6), ('E', 'F', 3)
])

mst = nx.minimum_spanning_tree(G, weight='weight')
print('MST edges:', list(mst.edges(data=True)))
print('Total MST weight:', mst.size(weight='weight'))

# 可视化
pos = nx.spring_layout(G, seed=42)
plt.figure(figsize=(8, 5))
nx.draw_networkx_nodes(G, pos, node_color='lightblue', node_size=1000)
nx.draw_networkx_labels(G, pos)
# 非 MST 边灰色
nx.draw_networkx_edges(G, pos, edge_color='lightgray', width=1)
# MST 边红色
nx.draw_networkx_edges(mst, pos, edge_color='red', width=2.5)
edge_labels = nx.get_edge_attributes(G, 'weight')
nx.draw_networkx_edge_labels(G, pos, edge_labels=edge_labels)
plt.title('Minimum Spanning Tree (red edges)')
plt.axis('off')
plt.show()

4. Hamiltonian 路径 / 圈 与 TSP

Hamiltonian 路径：经过图中每个顶点恰好一次的路径。Hamiltonian 圈：回到起点的闭合路径。TSP（旅行商问题）：在完全图中找最短的 Hamiltonian 圈。

TSP 数学表达

决策变量：xij ∈ {0, 1}，是否从 i 直接走到 j。

目标函数：

min Σi,j cij · xij

约束：

  • 每个点恰好进入一次：Σi xij = 1，∀j
  • 每个点恰好离开一次：Σj xij = 1，∀i
  • 消除子回路（subtour elimination），常用 MTZ 约束或 Miller-Tucker-Zemlin
takeaway：TSP 是 NP-hard。小规模可用整数规划精确求解；大规模用启发式（最近邻、2-opt、模拟退火）。 
import itertools
import numpy as np

# 小规模 TSP：暴力枚举
nodes = ['A', 'B', 'C', 'D']
# 对称距离矩阵
np.random.seed(42)
dist = np.array([
    [0, 10, 15, 20],
    [10, 0, 35, 25],
    [15, 35, 0, 30],
    [20, 25, 30, 0]
])

best = None
best_cost = float('inf')
for perm in itertools.permutations(range(1, len(nodes))):
    route = (0,) + perm + (0,)
    cost = sum(dist[route[i]][route[i+1]] for i in range(len(route)-1))
    if cost < best_cost:
        best_cost = cost
        best = route

print('Best route:', [nodes[i] for i in best])
print('Minimum distance:', best_cost)

# 用 NetworkX 的 TSP 近似（Christofides）
import networkx as nx
G = nx.complete_graph(len(nodes))
for i, j in itertools.combinations(range(len(nodes)), 2):
    G[i][j]['weight'] = dist[i][j]

approx = nx.approximation.traveling_salesman_problem(G, weight='weight', cycle=True)
print('Approx route:', [nodes[i] for i in approx])

5. 图着色 Graph Coloring

问题：给每个顶点分配一种“颜色”，使得相邻顶点颜色不同，且使用的颜色总数最少。应用：考试排期、化学品隔离、频率分配。

数学表达

决策变量：

  • cv ∈ {1, 2, ..., K}：顶点 v 的颜色编号
  • yk ∈ {0, 1}：颜色 k 是否被使用

目标函数：

min Σk=1K yk

约束：

  • 相邻顶点颜色不同：cu ≠ cv，∀(u, v) ∈ E
  • 若颜色 k 被使用，则 yk = 1
takeaway：最小着色数是 NP-hard。常用贪心算法或整数规划；实际问题常转化为“最少需要几个时间段”。 
import networkx as nx
import matplotlib.pyplot as plt

# 考试排期：课程为节点，有共同学生的课程连边
G = nx.Graph()
G.add_nodes_from(['Math', 'Physics', 'CS', 'English', 'History'])
G.add_edges_from([
    ('Math', 'Physics'), ('Math', 'CS'),
    ('Physics', 'English'), ('CS', 'History'),
    ('English', 'History')
])

# 贪心着色
coloring = nx.coloring.greedy_color(G, strategy='largest_first')
num_colors = max(coloring.values()) + 1
print('Color assignment:', coloring)
print('Minimum time slots needed (greedy):', num_colors)

# 可视化
colors = ['#fca5a5', '#93c5fd', '#86efac', '#fde047', '#d8b4fe']
node_colors = [colors[coloring[n] % len(colors)] for n in G.nodes]
pos = nx.spring_layout(G, seed=42)
plt.figure(figsize=(7, 5))
nx.draw(G, pos, with_labels=True, node_color=node_colors, node_size=1500, font_size=11)
plt.title('Exam Scheduling as Graph Coloring')
plt.axis('off')
plt.show()

6. 指派问题：Assignment / Bipartite Matching

问题：n 个工人完成 n 项任务，每人一项，每项一人，最小化总成本。这是二分图上的最小权完美匹配。

数学表达

决策变量：xij ∈ {0, 1}，工人 i 是否做任务 j。

目标函数：

min Σi=1n Σj=1n cij · xij

约束：

  • 每个工人一项任务：Σj xij = 1，∀i
  • 每项任务一个工人：Σi xij = 1，∀j
takeaway：指派问题是 LP 松弛后自动得到整数解的特殊情况，可用匈牙利算法或 scipy.optimize.linear_sum_assignment。 
import numpy as np
from scipy.optimize import linear_sum_assignment

# 成本矩阵：行 = 志愿者队伍，列 = 污染河段
cost = np.array([
    [9, 2, 7, 8],
    [6, 4, 3, 7],
    [5, 8, 1, 4],
    [7, 6, 9, 5]
])

row_ind, col_ind = linear_sum_assignment(cost)
print('Optimal assignment:')
for r, c in zip(row_ind, col_ind):
    print(f'  Team {r} → Section {c}, cost = {cost[r, c]}')
print('Total minimum cost:', cost[row_ind, col_ind].sum())

例题 1：跨境物流最短时间路径

9 个货运港口，双向道路运输，各有固定卡车通行时间。货物从起点 S1 到终点 T9，必须依次经过保税中转港 K3 和海关查验港 K6。求总通行时间最短的路线。

建模要点
  • 节点：9 个港口
  • 边：双向道路，权重 = 通行时间
  • 约束：路径必须 S1 → K3 → K6 → T9
  • 模型：分段 Dijkstra：S1→K3、K3→K6、K6→T9
Python 思路
path1 = nx.shortest_path(G, 'S1', 'K3', weight='weight')
path2 = nx.shortest_path(G, 'K3', 'K6', weight='weight')
path3 = nx.shortest_path(G, 'K6', 'T9', weight='weight')
total = (nx.shortest_path_length(G, 'S1', 'K3', weight='weight') +
         nx.shortest_path_length(G, 'K3', 'K6', weight='weight') +
         nx.shortest_path_length(G, 'K6', 'T9', weight='weight'))

例题 2：5G 光纤网络最小成本

9 个点，其中 5 个是核心 5G 基站，其余是中继塔。点间有双向光纤链接及安装成本。要求 5 个核心基站必须互相连通，中继塔可自由选择。求最小总安装成本。

建模要点
  • 节点：9 个点（5 核心 + 4 中继）
  • 边：双向光纤，权重 = 安装成本

8. 建模起手式：拿到题先填这 6 项

问题 示例
1. 节点 V 是什么？城市、建筑物、课程、人群、化学品
2. 边 E 是什么？道路、连接、冲突关系、依赖关系
3. 有向还是无向？道路双向用无向；物流/依赖用有向
4. 权重是什么？距离、时间、成本、交互次数
5. 约束是什么？必须经过某点、容量限制、不能同色
6. 优化目标是什么？最短、最小成本、最少颜色、最大覆盖
推荐库： networkx（图构建与经典算法）、 scipy.optimize.linear_sum_assignment（指派问题）、 pulp（整数规划，适合 TSP / 着色 / 复杂约束）。
