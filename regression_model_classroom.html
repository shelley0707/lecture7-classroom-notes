1. EDA — Exploratory Data Analysis

We are predicting final_math_score from student records. Before modeling, we inspect the data for shape, quality, and relationships.

  • Data overview: use df.shape, df.head(), df.info(), and df.describe() to inspect dimensions, types, and basic statistics.
  • Missing values: some students may not report sleep or workout hours; count missing entries per column.
  • Distribution plots: histograms with KDE curves reveal skewness and outliers in scores and habits.
  • Correlation heatmap: pairwise Pearson correlations help spot redundant predictors (e.g., prior math and physics scores may be correlated).
Takeaway: EDA tells us which predictors need imputation, which are categorical, and which may be collinear. 
import warnings
warnings.filterwarnings('ignore')

import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns

# -----------------------------
# Build a synthetic student dataset
# -----------------------------
np.random.seed(42)
n = 500

df = pd.DataFrame({
    'student_id': range(1, n + 1),
    'previous_math_score': np.random.normal(72, 12, n).clip(0, 100),
    'physics_score': np.random.normal(70, 11, n).clip(0, 100),
    'study_hours_last_3m': np.random.normal(120, 40, n).clip(0, 300),
    'workout_hours_last_3m': np.random.normal(30, 15, n).clip(0, 120),
    'avg_sleep_hours': np.random.normal(7, 1.2, n).clip(4, 11),
    'times_late_last_3m': np.random.poisson(4, n),
})

# favorite_subject is categorical
df['favorite_subject'] = np.random.choice(
    ['Math', 'Physics', 'Chemistry', 'Biology', 'English', 'History'],
    size=n,
    p=[0.25, 0.20, 0.15, 0.15, 0.15, 0.10]
)

# midterm_math is highly correlated with previous_math_score to demonstrate multicollinearity
df['midterm_math_score'] = (
    0.95 * df['previous_math_score']
    + np.random.normal(0, 3, n)
).clip(0, 100)

# Final math score: realistic combination of predictors
df['final_math_score'] = (
    0.35 * df['previous_math_score']
    + 0.20 * df['physics_score']
    + 0.15 * df['midterm_math_score']
    + 0.08 * df['study_hours_last_3m']
    + 0.05 * df['workout_hours_last_3m']
    + 2.0 * df['avg_sleep_hours']
    - 1.5 * df['times_late_last_3m']
    + np.where(df['favorite_subject'] == 'Math', 5, 0)
    + np.where(df['favorite_subject'] == 'Physics', 2, 0)
    + np.random.normal(0, 6, n)
).clip(0, 100)

# Inject missing values in continuous habit columns
for col, count in [('study_hours_last_3m', 25), ('avg_sleep_hours', 20), ('workout_hours_last_3m', 15)]:
    idx = np.random.choice(df.index, size=count, replace=False)
    df.loc[idx, col] = np.nan

print("Shape:", df.shape)

2. Preprocessing — ID, VIF, and Imputation

Clean the student data and prepare it for regression.

  • Drop ID column: student_id carries no predictive signal.
  • VIF (Variance Inflation Factor): detects multicollinearity. We drop the feature with the highest VIF iteratively until all VIFs are below 10.
  • Categorical encoding: favorite_subject is converted to dummy variables with drop_first=True to avoid the dummy-variable trap.
  • Median imputation: missing habit hours are filled with the column median.
Takeaway: Dropping redundant score columns and encoding the categorical subject keeps the regression coefficients stable and interpretable. 
from sklearn.impute import SimpleImputer

# -----------------------------
# 1. Drop ID column
# -----------------------------
df = df.drop(columns=['student_id'])
print("Dropped ID column. Remaining columns:", list(df.columns))

# -----------------------------
# 2. Encode categorical variable into dummies
# -----------------------------
df = pd.get_dummies(df, columns=['favorite_subject'], drop_first=True, dtype=int)
print("\nColumns after dummy encoding:", list(df.columns))

# -----------------------------
# 3. Iterative VIF removal
# -----------------------------
def compute_vif(X):
    """Compute VIF for each column using statsmodels OLS."""
    vif_data = pd.DataFrame()
    vif_data['Feature'] = X.columns
    vifs = []
    for col in X.columns:
        y = X[col].astype(float)
        X_others = sm.add_constant(X.drop(columns=[col]))
        model = sm.OLS(y, X_others).fit()
        vifs.append(1 / (1 - model.rsquared))
    vif_data['VIF'] = vifs
    return vif_data

numeric_cols = df.select_dtypes(include=[np.number]).columns.drop('final_math_score')
X_vif = df[numeric_cols].copy()
X_vif = X_vif.fillna(X_vif.median())

vif_threshold = 10.0
removed_features = []

while True:
    vif_data = compute_vif(X_vif)
    max_vif = vif_data['VIF'].max()
    if max_vif <= vif_threshold:
        break
    feature_to_remove = vif_data.loc[vif_data['VIF'].idxmax(), 'Feature']
    removed_features.append(feature_to_remove)
    X_vif = X_vif.drop(columns=[feature_to_remove])

print("\nVIF after iterative removal (threshold = 10):")
print(vif_data)
print(f"\nRemoved features: {removed_features}")

# Drop the same columns from the main dataframe
df = df.drop(columns=removed_features)

# -----------------------------
# 4. Median imputation
# -----------------------------
imputer = SimpleImputer(strategy='median')
feature_cols = df.columns.drop('final_math_score')
df[feature_cols] = imputer.fit_transform(df[feature_cols])

print("\nMissing values after imputation:")
print(df.isnull().sum())

3. Train-Test Split & Standard Scaling

Separate students into training and test sets, then scale numeric predictors.

  • 70/30 split: 70% of students for training, 30% for testing, fixed with random_state.
  • StandardScaler: transforms each numeric predictor to mean 0 and standard deviation 1.
  • Fit on train, transform both: compute scaling statistics from the training set only to avoid data leakage.
Takeaway: Scaling keeps units comparable (e.g., study hours vs. sleep hours) and improves numerical stability for OLS and regularization. 
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler

# -----------------------------
# 1. Separate features and target
# -----------------------------
X = df.drop(columns=['final_math_score'])
y = df['final_math_score']

# -----------------------------
# 2. 70/30 split
# -----------------------------
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.30, random_state=42
)
print(f"Train size: {X_train.shape[0]}")
print(f"Test size:  {X_test.shape[0]}")

# -----------------------------
# 3. StandardScaler: fit on train, transform both
# -----------------------------
scaler = StandardScaler()
X_train_scaled = pd.DataFrame(
    scaler.fit_transform(X_train),
    columns=X_train.columns,
    index=X_train.index
)
X_test_scaled = pd.DataFrame(
    scaler.transform(X_test),
    columns=X_test.columns,
    index=X_test.index
)

print("\nTrain set means (should be ~0):")
print(X_train_scaled.mean().round(3))
print("\nTrain set standard deviations (should be ~1):")
print(X_train_scaled.std().round(3))

4. Linear Regression with statsmodels OLS

Fit an OLS model to predict final math score and evaluate its performance.

  • statsmodels OLS: detailed summary with p-values, standard errors, and confidence intervals for each predictor.
  • R²: proportion of variance in final math score explained by the model.
  • Adjusted R²: penalizes R² for adding unnecessary predictors such as extra dummy variables.
  • RMSE: average prediction error in exam-score points.
  • Residual plot: predicted vs. residual to check heteroscedasticity and non-linearity.
Takeaway: OLS tells us which habits and prior scores are statistically significant predictors of final math performance. 
import statsmodels.api as sm
from sklearn.metrics import mean_squared_error, r2_score

# -----------------------------
# 1. Fit OLS with constant term
# -----------------------------
X_train_const = sm.add_constant(X_train_scaled)
X_test_const = sm.add_constant(X_test_scaled)

model_ols = sm.OLS(y_train, X_train_const).fit()
print(model_ols.summary())

# -----------------------------
# 2. Predict and compute metrics
# -----------------------------
y_train_pred = model_ols.predict(X_train_const)
y_test_pred = model_ols.predict(X_test_const)

r2_train = r2_score(y_train, y_train_pred)
r2_test = r2_score(y_test, y_test_pred)

n = X_train.shape[0]
p = X_train.shape[1]
adj_r2_train = 1 - (1 - r2_train) * (n - 1) / (n - p - 1)
adj_r2_test = 1 - (1 - r2_test) * (X_test.shape[0] - 1) / (X_test.shape[0] - p - 1)

rmse_train = np.sqrt(mean_squared_error(y_train, y_train_pred))
rmse_test = np.sqrt(mean_squared_error(y_test, y_test_pred))

print("\n--- Linear Regression Metrics (Final Math Score) ---")
print(f"Train R²:         {r2_train:.4f}")
print(f"Train Adj R²:     {adj_r2_train:.4f}")
print(f"Train RMSE:       {rmse_train:.2f} points")
print(f"Test  R²:         {r2_test:.4f}")
print(f"Test  Adj R²:     {adj_r2_test:.4f}")
print(f"Test  RMSE:       {rmse_test:.2f} points")

# -----------------------------
# 3. Residual plot
# -----------------------------
residuals = y_test - y_test_pred
plt.figure(figsize=(8, 5))
plt.scatter(y_test_pred, residuals, alpha=0.6, edgecolors='k')
plt.axhline(y=0, color='r', linestyle='--')
plt.xlabel('Predicted Values')
plt.ylabel('Residuals')
plt.title('Residual Plot (Test Set)')
plt.tight_layout()
plt.show()

5. Tuning — RidgeCV or Polynomial Features

Diagnose whether the exam-score model overfits or underfits, then adjust.

  • Overfitting: train R² is much higher than test R². Solution: RidgeCV (L2 regularization) to shrink less important predictors.
  • Underfitting: both R² values are low. Solution: generate 2nd-degree polynomial features (e.g., study_hours²) and select the best ones.
  • Balanced: if neither condition holds, keep the OLS model.
Takeaway: Regularization helps when we have many correlated predictors; polynomial terms help when relationships are curved. 
from sklearn.linear_model import RidgeCV
from sklearn.preprocessing import PolynomialFeatures
from sklearn.feature_selection import SelectKBest, f_regression

# -----------------------------
# Diagnose fit
# -----------------------------
gap = r2_train - r2_test
print(f"Train R²: {r2_train:.4f}")
print(f"Test  R²: {r2_test:.4f}")
print(f"Gap:      {gap:.4f}")

# -----------------------------
# Case 1: Overfitting -> RidgeCV
# -----------------------------
if gap > 0.10:
    print("\nDetected overfitting. Applying RidgeCV...")
    alphas = [0.001, 0.01, 0.1, 1, 10, 100, 1000]
    ridge = RidgeCV(alphas=alphas, cv=5)
    ridge.fit(X_train_scaled, y_train)

    y_train_ridge = ridge.predict(X_train_scaled)
    y_test_ridge = ridge.predict(X_test_scaled)

    print(f"Best alpha:       {ridge.alpha_}")
    print(f"Train R² (Ridge): {r2_score(y_train, y_train_ridge):.4f}")
    print(f"Test  R² (Ridge): {r2_score(y_test, y_test_ridge):.4f}")
    print(f"Test  RMSE:       {np.sqrt(mean_squared_error(y_test, y_test_ridge)):.2f} points")

# -----------------------------
# Case 2: Underfitting -> Polynomial features
# -----------------------------
elif r2_train < 0.50:
    print("\nDetected underfitting. Adding 2nd-degree polynomial features...")
    poly = PolynomialFeatures(degree=2, include_bias=False)
    X_train_poly = poly.fit_transform(X_train_scaled)
    X_test_poly = poly.transform(X_test_scaled)

    # Select top features to avoid explosion
    k = min(10, X_train_poly.shape[1])
    selector = SelectKBest(score_func=f_regression, k=k)
    X_train_selected = selector.fit_transform(X_train_poly, y_train)
    X_test_selected = selector.transform(X_test_poly)

    model_poly = sm.OLS(y_train, sm.add_constant(X_train_selected)).fit()
    y_test_poly = model_poly.predict(sm.add_constant(X_test_selected))

    print(f"Number of selected features: {k}")
    print(f"Test R² (Poly):  {r2_score(y_test, y_test_poly):.4f}")
    print(f"Test RMSE (Poly): {np.sqrt(mean_squared_error(y_test, y_test_poly)):.2f} points")

# -----------------------------
# Case 3: Balanced
# -----------------------------
else:
    print("\nModel looks balanced. Keep the OLS model.")

6. Construct Binary Target for Logistic Regression

Convert the continuous exam score into a pass/fail classification problem.

  • Median split: students scoring above the median final math score are labeled 1 (high performer), others 0. This creates a balanced two-class problem.
  • Re-split: because the target changed, generate a fresh train-test split and re-scale the features.
  • Stratify: preserve the high/low performer ratio in both train and test sets.
Takeaway: Binarization lets us reuse the same predictors for logistic regression; the threshold should reflect a meaningful cutoff (e.g., passing grade) in real use. 
# -----------------------------
# 1. Median split of final math score
# -----------------------------
median_score = y.median()
print(f"Median final math score: {median_score:.2f}")

y_binary = (y > median_score).astype(int)
print(f"\nClass distribution (1 = above median, 0 = below median):")
print(y_binary.value_counts())

# -----------------------------
# 2. Re-split for classification
# -----------------------------
X_train_clf, X_test_clf, y_train_clf, y_test_clf = train_test_split(
    X, y_binary,
    test_size=0.30,
    random_state=42,
    stratify=y_binary
)
print(f"\nClassification train size: {X_train_clf.shape[0]}")
print(f"Classification test size:  {X_test_clf.shape[0]}")

# -----------------------------
# 3. Re-scale
# -----------------------------
scaler_clf = StandardScaler()
X_train_clf_scaled = scaler_clf.fit_transform(X_train_clf)
X_test_clf_scaled = scaler_clf.transform(X_test_clf)

print("\nTrain class balance:")
print(y_train_clf.value_counts(normalize=True).round(3))

7. Logistic Regression — Evaluation & ROC

Train a classifier to predict whether a student scores above the median in math.

  • Confusion matrix: counts of correctly and incorrectly classified high/low performers.
  • Classification report: precision, recall, and F1-score for each performance group.
  • ROC curve: trade-off between identifying high performers and falsely flagging low performers.
  • AUC: overall discriminative ability; 1 is perfect, 0.5 is random guessing.
  • Youden index: TPR - FPR; finds the probability threshold that best balances sensitivity and specificity.
Takeaway: For student interventions, you may prefer high recall (catch struggling students) or high precision (avoid false alarms); Youden gives a balanced cutoff. 
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import (
    confusion_matrix, classification_report,
    roc_curve, auc
)

# -----------------------------
# 1. Fit logistic regression
# -----------------------------
log_reg = LogisticRegression(max_iter=1000)
log_reg.fit(X_train_clf_scaled, y_train_clf)

# -----------------------------
# 2. Predict and report
# -----------------------------
y_prob = log_reg.predict_proba(X_test_clf_scaled)[:, 1]
y_pred = log_reg.predict(X_test_clf_scaled)

print("Confusion Matrix:")
print(confusion_matrix(y_test_clf, y_pred))

print("\nClassification Report:")
print(classification_report(y_test_clf, y_pred))

# -----------------------------
# 3. ROC curve and AUC
# -----------------------------
fpr, tpr, thresholds = roc_curve(y_test_clf, y_prob)
roc_auc = auc(fpr, tpr)

plt.figure(figsize=(8, 6))
plt.plot(
    fpr, tpr,
    color='darkorange', lw=2,
    label=f'ROC curve (AUC = {roc_auc:.4f})'
)
plt.plot([0, 1], [0, 1], color='navy', lw=2, linestyle='--')
plt.xlabel('False Positive Rate')
plt.ylabel('True Positive Rate')
plt.title('ROC Curve')
plt.legend(loc='lower right')
plt.grid(True, alpha=0.3)
plt.tight_layout()
plt.show()

# -----------------------------
# 4. Youden index -> best threshold
# -----------------------------
youden_index = tpr - fpr
best_idx = np.argmax(youden_index)
best_threshold = thresholds[best_idx]

print(f"\nBest threshold (Youden): {best_threshold:.4f}")
print(f"TPR at best threshold:   {tpr[best_idx]:.4f}")
print(f"FPR at best threshold:   {fpr[best_idx]:.4f}")
print(f"Youden index value:      {youden_index[best_idx]:.4f}")
