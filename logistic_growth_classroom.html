1. What is Logistic Growth?

Exponential growth assumes unlimited resources: the bigger the population, the faster it grows. In reality, growth slows as the population approaches a ceiling called the carrying capacity.

  • Exponential: growth rate stays constant; curve shoots upward forever.
  • Logistic: growth rate shrinks as the population nears its maximum; curve becomes S-shaped.
  • Carrying capacity (K or L): the upper limit the environment or market can support.
Takeaway: Use logistic growth when you expect an initial rapid increase followed by saturation. 
import warnings
warnings.filterwarnings('ignore')

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt

# -----------------------------
# Compare exponential vs. logistic growth
# -----------------------------
t = np.linspace(0, 20, 200)
r = 0.3
K = 1000
N0 = 10

# Exponential: N(t) = N0 * exp(r*t)
exp_growth = N0 * np.exp(r * t)

# Logistic: N(t) = K / (1 + ((K-N0)/N0) * exp(-r*t))
logistic_growth = K / (1 + ((K - N0) / N0) * np.exp(-r * t))

plt.figure(figsize=(12, 5))
plt.plot(t, exp_growth, label='Exponential Growth', color='coral', linewidth=2)
plt.plot(t, logistic_growth, label='Logistic Growth', color='steelblue', linewidth=2)
plt.axhline(y=K, color='green', linestyle='--', label=f'Carrying Capacity K={K}')
plt.title('Exponential vs. Logistic Growth')
plt.xlabel('Time')
plt.ylabel('Population / Users')
plt.legend()
plt.grid(True, alpha=0.3)
plt.ylim(0, 1500)
plt.tight_layout()
plt.show()

2. The Logistic Function

The logistic model can be written as a differential equation or as its closed-form solution.

dN/dt = r · N · (1 - N/K)

Solving this gives the S-curve formula:

N(t) = K / (1 + A · e-rt)

where A = (K - N₀) / N₀ depends on the initial value.

An equivalent pre-calculus form uses the midpoint x₀:

f(x) = L / (1 + e-k(x - x₀))

  • L or K: carrying capacity (maximum value).
  • r or k: growth rate; larger = steeper rise.
  • x₀ or t₀: time at the inflection point (halfway to K).
Takeaway: Three parameters control the curve: capacity, growth rate, and midpoint timing. 
# -----------------------------
# Logistic function implementations
# -----------------------------
def logistic_solution(t, K, r, N0):
    """Closed-form logistic growth."""
    A = (K - N0) / N0
    return K / (1 + A * np.exp(-r * t))

def logistic_midpoint(x, L, k, x0):
    """Pre-calculus form centered at midpoint x0."""
    return L / (1 + np.exp(-k * (x - x0)))

# Example parameters
t = np.linspace(0, 40, 200)
K, r, N0 = 5000, 0.15, 100

# The two forms are equivalent if x0 = ln(A) / r
A = (K - N0) / N0
x0 = np.log(A) / r
print(f"A = {A:.4f}")
print(f"Equivalent midpoint x0 = {x0:.4f}")

# Verify equivalence
y1 = logistic_solution(t, K, r, N0)
y2 = logistic_midpoint(t, K, r, x0)
print(f"Max absolute difference: {np.max(np.abs(y1 - y2)):.2e}")

plt.figure(figsize=(12, 5))
plt.plot(t, y1, label='N(t) = K / (1 + A·e^(-rt))', color='steelblue', linewidth=2)
plt.axhline(y=K, color='green', linestyle='--', alpha=0.7, label=f'K = {K}')
plt.axvline(x=x0, color='orange', linestyle='--', alpha=0.7, label=f'Midpoint x0 = {x0:.2f}')
plt.title('Logistic Growth Curve')
plt.xlabel('Time')
plt.ylabel('Users / Population')
plt.legend()
plt.grid(True, alpha=0.3)
plt.tight_layout()
plt.show()

3. Real-World Applications

Any process with a natural ceiling can be modeled with logistic growth.

  • Biology & ecology: bacterial or animal populations limited by food and space.
  • Marketing: product adoption from early adopters to mainstream users.
  • Epidemiology: disease spread where K is the total susceptible population.
  • Technology: user-base growth of a new app or platform.
Takeaway: Identify the carrying capacity first; it is often set by market size, resources, or population limits. 
# -----------------------------
# Application: app user adoption
# -----------------------------
weeks = np.arange(0, 53)
K = 50_000       # total addressable market
r = 0.25         # weekly growth rate
N0 = 200         # initial users

users = logistic_solution(weeks, K, r, N0)
np.random.seed(42)
users_noisy = users + np.random.normal(0, 800, len(weeks))
users_noisy = np.maximum(users_noisy, 0)

df = pd.DataFrame({'week': weeks, 'users': users_noisy.astype(int)})
print(df.head(10))

plt.figure(figsize=(12, 5))
plt.scatter(df['week'], df['users'], alpha=0.6, color='steelblue', label='Observed users')
plt.plot(weeks, users, color='green', linewidth=2, label='True logistic curve')
plt.axhline(y=K, color='red', linestyle='--', alpha=0.7, label=f'Market capacity K={K:,}')
plt.title('App User Adoption: Logistic Growth Example')
plt.xlabel('Week')
plt.ylabel('Weekly Active Users')
plt.legend()
plt.grid(True, alpha=0.3)
plt.tight_layout()

4. Method 1 — Fit with Full Data

When you have many historical observations, use non-linear least squares to estimate K, r, and N₀ directly from the data.

  • Curve fitting: minimize the squared distance between the model and observed points.
  • Initial guesses: K ≈ final plateau, r ≈ early growth rate, N₀ ≈ first observation.
  • Goodness of fit: report R² to see how well the curve captures the trend.
Takeaway: With enough data, curve fitting gives the most accurate parameter estimates and uncertainty bounds. 
from scipy.optimize import curve_fit

# -----------------------------
# Fit logistic curve to full data
# -----------------------------
def logistic_fit(t, K, r, N0):
    return K / (1 + ((K - N0) / N0) * np.exp(-r * t))

# Use the noisy app data from Module 3
p0 = [60_000, 0.2, 100]  # initial guess for [K, r, N0]
bounds = ([0, 0, 0], [200_000, 2, 10_000])

popt, pcov = curve_fit(logistic_fit, df['week'], df['users'], p0=p0, bounds=bounds)
K_fit, r_fit, N0_fit = popt

print(f"Fitted K  = {K_fit:,.2f}")
print(f"Fitted r  = {r_fit:.4f}")
print(f"Fitted N0 = {N0_fit:.2f}")

# R-squared
y_pred = logistic_fit(df['week'], *popt)
ss_res = np.sum((df['users'] - y_pred) ** 2)
ss_tot = np.sum((df['users'] - df['users'].mean()) ** 2)
r2 = 1 - ss_res / ss_tot
print(f"R²        = {r2:.4f}")

# Plot fit
plt.figure(figsize=(12, 5))
plt.scatter(df['week'], df['users'], alpha=0.6, color='steelblue', label='Observed')
plt.plot(df['week'], y_pred, color='purple', linewidth=2, label='Fitted logistic curve')
plt.axhline(y=K_fit, color='red', linestyle='--', alpha=0.7, label=f'Fitted K = {K_fit:,.0f}')
plt.title('Logistic Curve Fit to App User Data')
plt.xlabel('Week')
plt.ylabel('Users')
plt.legend()
plt.grid(True, alpha=0.3)
plt.tight_layout()
plt.show()

5. Method 2 — Estimate r with Limited Early Data

When only a few early observations are available, assume the series is still approximately exponential and estimate r before fitting the full logistic curve.

  • Doubling time Td: if the population doubles every Td periods, r ≈ ln(2) / Td.
  • Known percentage growth s per period: r = ln(1 + s).
  • Two early points: r ≈ ln(N₂/N₁) / (t₂ - t₁).
  • Benchmark: use industry reports or reference data.
Takeaway: Early estimates of r provide a starting guess for K and speed up logistic fitting when data is sparse. 
# -----------------------------
# Estimate r from early observations
# -----------------------------
# Suppose we only know the first 8 weeks from the app data
early = df.iloc[:8].copy()
print("Early observations:")
print(early)

# Method A: two-point estimate
N1, t1 = early.iloc[0]['users'], early.iloc[0]['week']
N2, t2 = early.iloc[-1]['users'], early.iloc[-1]['week']
r_two_point = np.log(N2 / N1) / (t2 - t1)
print(f"\nTwo-point r estimate: {r_two_point:.4f}")

# Method B: regression on log(users) vs. time
early['log_users'] = np.log(early['users'])
coef = np.polyfit(early['week'], early['log_users'], 1)
r_log_linear = coef[0]
print(f"Log-linear r estimate: {r_log_linear:.4f}")

# Method C: known 15.5% weekly growth
s = 0.155
r_pct = np.log(1 + s)
print(f"From 15.5% weekly growth: r = ln(1.155) = {r_pct:.4f}")

# Compare with true r
print(f"True r used to generate data: {r:.4f}")

6. Worked Example: Small Dataset

The slide provides a small dataset with carrying capacity K = 5,000 and early values from week 0 to week 7. We fit the logistic model and compare it with the Desmos result.

  • Data: 8 early observations starting at 100.
  • Assumed K: 5,000 (set by market research).
  • Fit: fix K and estimate r and N₀, or estimate all three.
Takeaway: Even with only a few points, a reasonable K assumption lets you project the full S-curve. 
# -----------------------------
# Reproduce the slide's small dataset
# -----------------------------
slide_data = pd.DataFrame({
    'week': [0, 1, 2, 3, 4, 5, 6, 7],
    'users': [100.0, 115.81, 134.05, 155.07, 179.26,
              207.07, 238.98, 275.53]
})
print(slide_data)

# Fit with K fixed at 5000
def logistic_fixed_K(t, r, N0):
    K = 5000
    return K / (1 + ((K - N0) / N0) * np.exp(-r * t))

popt_fixed, _ = curve_fit(logistic_fixed_K, slide_data['week'], slide_data['users'],
                          p0=[0.15, 100], bounds=([0, 0], [2, 5000]))
r_fixed, N0_fixed = popt_fixed
print(f"\nWith K=5000 fixed:")
print(f"  r  = {r_fixed:.4f}")
print(f"  N0 = {N0_fixed:.2f}")

# Equivalent Desmos-style equation: y = 5000 / (1 + e^(-(0.15*x - 3.89182)))
# Our fitted form can be rewritten similarly
A_fixed = (5000 - N0_fixed) / N0_fixed
x0_fixed = np.log(A_fixed) / r_fixed
print(f"  Equivalent: y = 5000 / (1 + e^(-({r_fixed:.4f}*x - {r_fixed*x0_fixed:.4f})))")

# Plot
weeks_extended = np.linspace(0, 50, 200)
plt.figure(figsize=(12, 5))
plt.scatter(slide_data['week'], slide_data['users'], color='steelblue', s=80, label='Observed data')
plt.plot(weeks_extended, logistic_fixed_K(weeks_extended, *popt_fixed),
         color='purple', linewidth=2, label='Fitted logistic (K=5000)')
plt.axhline(y=5000, color='red', linestyle='--', alpha=0.7, label='K = 5,000')
plt.title('Worked Example: Small-Dataset Logistic Fit')
plt.xlabel('Week')
plt.ylabel('Users')
plt.legend()
plt.grid(True, alpha=0.3)
plt.tight_layout()
plt.show()

7. Forecast Future Values

Once the model is fitted, plug future time points into the logistic function to obtain forecasts and a saturation horizon.

  • Point forecast: predicted value at each future week.
  • Saturation: when does the curve reach, say, 95% of K?
  • Visual check: overlay forecasts on historical data.
Takeaway: Logistic forecasts naturally level off; they do not blow up like exponential projections. 
# -----------------------------
# Forecast using the worked-example model
# -----------------------------
future_weeks = np.arange(8, 21)
forecast = logistic_fixed_K(future_weeks, *popt_fixed)
forecast_df = pd.DataFrame({'week': future_weeks, 'forecast_users': forecast.round(2)})
print(forecast_df)

# Time to reach 95% of K
K_target = 0.95 * 5000
# Solve K / (1 + A*e^(-rt)) = 0.95K  =>  t = (ln(19) + ln(A)) / r
time_to_95 = (np.log(19) + np.log(A_fixed)) / r_fixed
print(f"\nTime to reach 95% of K (4,750 users): week {time_to_95:.1f}")

# Plot history + forecast
all_weeks = np.concatenate([slide_data['week'], future_weeks])
all_values = np.concatenate([slide_data['users'], forecast])

plt.figure(figsize=(12, 5))
plt.scatter(slide_data['week'], slide_data['users'], color='steelblue', s=80, label='Observed')
plt.plot(future_weeks, forecast, color='purple', linewidth=2, linestyle='--', label='Forecast')
plt.plot(all_weeks, logistic_fixed_K(all_weeks, *popt_fixed), color='gray', alpha=0.5, label='Full curve')
plt.axhline(y=5000, color='red', linestyle='--', alpha=0.7, label='K = 5,000')
plt.axhline(y=K_target, color='orange', linestyle=':', alpha=0.7, label='95% of K')
plt.title('Logistic Growth Forecast')
plt.xlabel('Week')
plt.ylabel('Users')
plt.legend()
plt.grid(True, alpha=0.3)
plt.tight_layout()
plt.show()

8. Summary

Logistic growth is the go-to model when growth must eventually slow down.

  • Shape: S-curve from slow start → rapid growth → saturation.
  • Key parameters: K (capacity), r (growth rate), N₀ or x₀ (timing).
  • Full data: non-linear least squares fits all parameters.
  • Sparse data: estimate r from early exponential behavior and fix K from domain knowledge.
Takeaway: Always ask: is there a hard ceiling? If yes, logistic growth is likely more realistic than pure exponential growth.
