1. 概率分布是随机性的积木

在建模前，先要判断随机变量适合用什么分布描述。PPT 中提到的核心分布如下：

分布适用场景Python
Normal身高、误差、任务耗时np.random.normal(μ, σ)
Binomialn 次独立成功次数np.random.binomial(n, p)
Poisson单位时间事件发生次数np.random.poisson(λ)
Uniform等可能取值np.random.uniform(a, b)
Exponential事件间隔时间 / 寿命np.random.exponential(1/λ)
takeaway：选分布 = 判断变量是连续/离散、是否有上界、事件是否独立。

2. Monte Carlo 案例：飘落的叶子

场景：一片叶子从 5 米高的树枝脱落。水平风速 V ~ Normal(μ=5, σ=3) m/s，风向 θ 可在一定范围内随机摆动，垂直下落速度恒为 1 m/s。求叶子落地点在地面上的二维分布。

模型设定

已知参数：

  • h = 5 m：初始高度
  • vvertical = 1 m/s：垂直下落速度
  • V ~ N(5, 3)：水平风速
  • θ ~ N(μdir, σdir)：风向，单位弧度（可手动调整方向范围）

核心变量：

  • T = h / vvertical = 5 s：下落时间
  • X = V · T · cos(θ)：落点 x 坐标
  • Y = V · T · sin(θ)：落点 y 坐标
  • R = √(X² + Y²)：落点与树干的水平距离

扩展：通过调整 μdir 和 σdir，可以模拟"主要从东边吹来但有一定扩散"的风。

takeaway：一个简单的随机输入（风速）乘以一个确定的时间，就得到随机输出（落点）。 
import numpy as np
import matplotlib.pyplot as plt

# Fix the random seed for reproducible results.
np.random.seed(42)
# Number of simulated leaves.
n = 10000

# --- Physical parameters ---
h = 5.0                    # Initial falling height in meters.
v_vert = 1.0               # Constant vertical falling speed in m/s.
mu_v = 5.0                 # Mean horizontal wind speed in m/s.
sigma_v = 3.0              # Standard deviation of wind speed in m/s.
# Wind direction: mean direction (radians) and angular spread (radians).
# 0 radians = positive x-axis; π/2 radians = positive y-axis.
mu_dir = np.pi / 6         # Mean wind direction: 30 degrees.
sigma_dir = np.pi / 4      # Direction spread: ±45 degrees.

# --- Monte Carlo sampling ---
# Fall time is deterministic because vertical speed is constant.
T = h / v_vert
# Sample wind speed and direction for each leaf.
V = np.random.normal(mu_v, sigma_v, n)
theta = np.random.normal(mu_dir, sigma_dir, n)
# Convert polar (speed, direction) to Cartesian landing coordinates (X, Y).
X = V * T * np.cos(theta)
Y = V * T * np.sin(theta)
# Total horizontal displacement from the drop point.
R = np.sqrt(X**2 + Y**2)

# --- Summary statistics ---
print(f'Fall time: {T:.1f} s')
print(f'Mean landing X: {X.mean():.2f} m')
print(f'Mean landing Y: {Y.mean():.2f} m')
print(f'Mean horizontal distance: {R.mean():.2f} m')
print(f'P(landing within 40 m): {(R <= 40).mean():.2%}')

# --- Visualizations ---
fig, axes = plt.subplots(2, 2, figsize=(12, 10))
# 1. Histogram of sampled wind speeds.
axes[0, 0].hist(V, bins=40, color='skyblue', edgecolor='white')
axes[0, 0].set_title('Wind Speed Distribution V ~ N(5, 3)')
axes[0, 0].set_xlabel('Wind speed (m/s)')
axes[0, 0].set_ylabel('Count')

# 2. Histogram of total horizontal landing distance.
axes[0, 1].hist(R, bins=40, color='seagreen', edgecolor='white')
axes[0, 1].axvline(R.mean(), color='red', linestyle='--', label=f'Mean={R.mean():.1f}m')
axes[0, 1].set_title('Landing Distance Distribution')
axes[0, 1].set_xlabel('Horizontal distance (m)')
axes[0, 1].set_ylabel('Count')
axes[0, 1].legend()

# 3. Top-down scatter plot of landing positions.
axes[1, 0].scatter(X, Y, s=5, alpha=0.3, c='green')
axes[1, 0].scatter(0, 0, c='red', s=100, marker='x', label='Tree base')
axes[1, 0].set_aspect('equal')
axes[1, 0].set_title('Top-Down View of Landing Positions')
axes[1, 0].set_xlabel('X (m)')
axes[1, 0].set_ylabel('Y (m)')
axes[1, 0].legend()

# 4. 2D density heatmap of landing positions.
axes[1, 1].hist2d(X, Y, bins=40, cmap='Greens')
axes[1, 1].scatter(0, 0, c='red', s=100, marker='x')
axes[1, 1].set_aspect('equal')
axes[1, 1].set_title('Landing Density Heatmap')
axes[1, 1].set_xlabel('X (m)')
axes[1, 1].set_ylabel('Y (m)')

plt.tight_layout()
plt.show()

3. Monte Carlo 案例：估算 π

思想：向边长为 2 的正方形内随机撒点，其中落在内切圆内的比例 ≈ 圆面积 / 正方形面积 = π/4。

数学表达

随机变量：(X, Y) ~ Uniform(-1, 1) × Uniform(-1, 1)

指示变量：

I = 1 if X² + Y² ≤ 1, else 0

估计量：

π̂ = 4 · (ΣI) / N

收敛：当 N → ∞，π̂ → π（大数定律）。

takeaway：用随机实验估算确定值，是 Monte Carlo 最直观的演示。 
import numpy as np
import matplotlib.pyplot as plt

# Fix the random seed for reproducibility.
np.random.seed(42)
# Total number of random points thrown inside the square.
N = 5000
# Sample (X, Y) uniformly from the square [-1, 1] x [-1, 1].
x = np.random.uniform(-1, 1, N)
y = np.random.uniform(-1, 1, N)
# Indicator: point falls inside the unit circle.
inside = x**2 + y**2 <= 1

# Estimate π as 4 * (fraction of points inside the circle).
pi_est = 4 * inside.sum() / N
print(f'Estimated π with N={N}: {pi_est:.5f}')
print(f'Error: {abs(np.pi - pi_est):.5f}')

# --- Visualizations ---
fig, axes = plt.subplots(1, 2, figsize=(12, 5))
# 1. Scatter plot showing which points land inside vs outside the circle.
axes[0].scatter(x[inside], y[inside], c='blue', s=5, alpha=0.5, label='Inside circle')
axes[0].scatter(x[~inside], y[~inside], c='orange', s=5, alpha=0.5, label='Outside circle')
theta = np.linspace(0, 2*np.pi, 100)
axes[0].plot(np.cos(theta), np.sin(theta), 'r-', linewidth=2)
axes[0].set_aspect('equal')
axes[0].set_title(f'Estimate π ≈ {pi_est:.4f}')
axes[0].legend()

# 2. Convergence curve: estimate π with increasing sample sizes.
Ns = np.logspace(2, 5, 50, dtype=int)
ests = []
for n in Ns:
    xx = np.random.uniform(-1, 1, n)
    yy = np.random.uniform(-1, 1, n)
    ests.append(4 * (xx**2 + yy**2 <= 1).sum() / n)
axes[1].plot(Ns, ests, label='MC estimate')
axes[1].axhline(np.pi, color='red', linestyle='--', label='True π')
axes[1].set_xscale('log')
axes[1].set_title('Convergence of π Estimate')
axes[1].set_xlabel('Number of samples')
axes[1].set_ylabel('π estimate')
axes[1].legend()
plt.tight_layout()
plt.show()

4. Monte Carlo Simulation 的基本结构

Monte Carlo 不是“随机硬试”，而是一种数值实验：把不确定输入交给概率分布，通过大量重复抽样，让统计结果逼近真实答案。只要能把问题写成“输入 → 模型 → 输出”的形式，就可以用 Monte Carlo。

四步骨架
  1. 定义映射 y = g(X)：明确输入 X 和要估计的输出 y。例如 π 估计中，X = (x, y) 是随机点，y = 1圆内。
  2. 给随机输入选分布 X ~ F：根据实际背景选择 Normal、Uniform、Poisson 等，并确定参数。
  3. 重复 N 次试验：每次抽样 Xᵢ，计算 yᵢ = g(Xᵢ)，把结果记录下来。
  4. 汇总与诊断：算均值、概率、分位数、置信区间，并观察随 N 增大的收敛曲线。
核心公式

μ̂ = (1/N) Σᵢ g(Xᵢ)

σ̂² = (1/(N−1)) Σᵢ (g(Xᵢ) − μ̂)²

95% CI ≈ μ̂ ± 1.96 · σ̂ / √N

相对误差 ≈ 1 / √N（方差减半 → 误差只降 √2 倍）

大数定律保证 N → ∞ 时 μ̂ → μ；中心极限定理说明估计误差近似正态分布，从而能算置信区间。

takeaway：Monte Carlo 的强大之处在于：不关心 g(·) 多复杂，只要会抽样、会计算，就能得到带置信区间的近似解。 
import numpy as np
import matplotlib.pyplot as plt

# --- Generic Monte Carlo skeleton ---
def monte_carlo_estimate(sampler, model, N, seed=None):
    """
    sampler: callable that returns one random input X.
    model:   callable that maps X -> scalar output y.
    N:       number of Monte Carlo trials.
    """
    if seed is not None:
        np.random.seed(seed)
    results = np.empty(N)
    for i in range(N):
        X = sampler()
        results[i] = model(X)
    mean = results.mean()
    std = results.std(ddof=1)
    se = std / np.sqrt(N)
    ci_low, ci_high = mean - 1.96*se, mean + 1.96*se
    return mean, std, (ci_low, ci_high), results

# --- Example: estimate E[max(U1, U2)] where U ~ Uniform(0,1) ---
# Analytic answer = 2/3.
sampler = lambda: np.random.uniform(0, 1, size=2)
model   = lambda U: np.max(U)

N = 10000
est, std, (lo, hi), _ = monte_carlo_estimate(sampler, model, N, seed=42)
print(f'Estimate: {est:.5f}  (true = {2/3:.5f})')
print(f'Std dev : {std:.5f}')
print(f'95% CI  : [{lo:.5f}, {hi:.5f}]')

# --- Convergence check: estimate with increasing N ---
Ns = np.logspace(2, 4, 30, dtype=int)
means = []
for n in Ns:
    _, _, _, res = monte_carlo_estimate(sampler, model, n, seed=42)
    means.append(res.mean())

fig, ax = plt.subplots(figsize=(8, 4))
ax.plot(Ns, means, label='MC estimate')
ax.axhline(2/3, color='red', linestyle='--', label='True 2/3')
ax.set_xscale('log')
ax.set_xlabel('Number of trials N')
ax.set_ylabel('Estimate')
ax.set_title('Monte Carlo Convergence: E[max(U1,U2)]')
ax.legend(); plt.tight_layout(); plt.show()

5. Monte Carlo 案例：产品上线时间风险（CPM / PERT）

场景：一个科技初创公司准备发布旗舰产品。各任务持续时间是随机变量，需要估计总完成时间、30 天内上线概率、关键风险点。

任务与依赖（AoN 网络图）

上图是 AoN（Activity on Node）网络图，节点 = 任务，箭头 = 依赖关系。

五条并行线路
线路路径累计耗时
1A → C → EA + C + E
2B → D → EB + D + E
3A → GA + G
4B → GB + G
5FF

关键路径（Critical Path）：所有并行线路中的最长者决定 H 的开始时间。

T = max(A+C+E, B+D+E, A+G, B+G, F) + H

方法背景：

  • CPM（Critical Path Method）：当任务时间确定时，找出关键路径。
  • PERT（Program Evaluation and Review Technique）：当任务时间是随机变量时，用 Monte Carlo 估计工期分布。
takeaway：先画 AoN 图找出所有并行线路，再用 Monte Carlo 模拟每条线路的随机耗时，最后用 max(...) 得到总工期分布。 
import numpy as np
import matplotlib.pyplot as plt
import networkx as nx

# Fix the random seed for reproducibility.
np.random.seed(42)
# Number of Monte Carlo trials.
n = 10000

# --- Sample task durations for each trial ---
A = np.random.normal(10, 2, n)      # Feature dev
B = np.random.normal(7, 1.5, n)     # UI/UX
C = np.random.normal(8, 2, n)       # Backend (depends on A)
D = np.random.normal(6, 1, n)       # Frontend (depends on B)
E = np.random.uniform(5, 10, n)     # Testing (depends on C, D)
F = np.random.normal(12, 3, n)      # Marketing
G = np.random.uniform(3, 7, n)      # Legal (depends on A, B)
H = 1                               # Launch (depends on E, F, G)

# --- Compute the five parallel paths in the project network ---
path1 = A + C + E
path2 = B + D + E
path3 = A + G
path4 = B + G
path5 = F
path_names = ['A+C+E', 'B+D+E', 'A+G', 'B+G', 'F']

# Identify the critical path for each trial (the longest path).
paths = np.vstack([path1, path2, path3, path4, path5])
critical_idx = np.argmax(paths, axis=0)
critical_counts = np.bincount(critical_idx, minlength=5)
critical_freq = critical_counts / n

# Total completion time = longest path + final launch task H.
T = np.max(paths, axis=0) + H

# --- Summary statistics ---
print(f'Mean completion time: {T.mean():.1f} days')
print(f'P(finish within 30 days): {(T <= 30).mean():.2%}')
print(f'P(finish within 35 days): {(T <= 35).mean():.2%}')
print(f'95% confidence interval: [{np.percentile(T, 2.5):.1f}, {np.percentile(T, 97.5):.1f}]')
print('\nCritical path frequency:')
for name, freq in zip(path_names, critical_freq):
    print(f'  {name}: {freq:.2%}')

# --- Visualizations ---
fig, axes = plt.subplots(1, 3, figsize=(18, 5))

# 1. Draw the Activity-on-Node (AoN) project network.
G_nx = nx.DiGraph()
edges = [('START','A'), ('START','B'), ('START','F'),
         ('A','C'), ('A','G'), ('B','D'), ('B','G'),
         ('C','E'), ('D','E'), ('E','H'), ('G','H'), ('F','H'), ('H','END')]
G_nx.add_edges_from(edges)

pos = {
    'START': (0, 0), 'A': (1, 1), 'B': (1, -1), 'F': (1, -2.5),
    'C': (2, 1.5), 'D': (2, -1.5), 'G': (2, 0),
    'E': (3, 0), 'H': (4, 0), 'END': (5, 0)
}
nx.draw_networkx_nodes(G_nx, pos, node_color='lightblue', node_size=1500, ax=axes[0])
nx.draw_networkx_edges(G_nx, pos, arrows=True, arrowsize=20, ax=axes[0],
                       edge_color='gray', node_size=1500)
nx.draw_networkx_labels(G_nx, pos, font_size=10, ax=axes[0])
axes[0].set_title('AoN Project Network (Activity on Node)')
axes[0].axis('off')

# 2. Histogram of total completion time with deadline line.
axes[1].hist(T, bins=50, color='coral', edgecolor='white')
axes[1].axvline(30, color='red', linestyle='--', label='30-day deadline')
axes[1].set_title('Distribution of Total Completion Time')
axes[1].set_xlabel('Days')
axes[1].set_ylabel('Frequency')
axes[1].legend()

# 3. Bar chart of how often each path is the critical path.
axes[2].barh(path_names, critical_freq, color='steelblue')
axes[2].set_title('Critical Path Frequency (PERT / Monte Carlo)')
axes[2].set_xlabel('Probability of being critical')
axes[2].set_xlim(0, 1)

plt.tight_layout()
plt.show()

6. Agent-Based Simulation：森林大火与 Schelling 隔离模型（由浅入深）

核心思想：Agent-Based Simulation（ABS）从“局部规则”出发，让大量个体（或细胞）按规则演化，观察宏观层面的涌现现象。下面用森林大火和 Schelling 种族隔离模型两个经典场景，逐步加深场景与参数难度；每个场景都配静态帧、参数扫描和动画代码。

6.1 森林大火模型（Forest Fire）

Block 1：确定性火源扩散

场景：在网格森林里，中心一棵树立刻着火；只要某个树的 8 邻域有火，下一时间步它必定被点燃。没有新树生长，也没有随机起火。

细胞状态：EMPTY = 0, TREE = 1, BURNING = 2, ASH = 3

规则：

  • BURNING → ASH
  • TREE 且邻居有 BURNING → BURNING
  • 其他状态不变

观察目标：火向外传播的“波前”是什么形状？边界碰到网格边缘后会怎样？

import numpy as np
import matplotlib.pyplot as plt
from matplotlib.colors import ListedColormap

EMPTY, TREE, BURNING, ASH = 0, 1, 2, 3
CMAP = ListedColormap(['white', 'green', 'red', 'black'])

def init_forest(grid_size, density):
    forest = np.zeros((grid_size, grid_size), dtype=int)
    forest[np.random.rand(grid_size, grid_size) < density] = TREE
    forest[grid_size//2, grid_size//2] = BURNING
    return forest

def step_fire_deterministic(forest):
    new = forest.copy()
    burn = (forest == BURNING)
    new[burn] = ASH
    # any TREE next to a burning cell catches fire
    for dr in (-1, 0, 1):
        for dc in (-1, 0, 1):
            if dr == 0 and dc == 0:
                continue
            shifted = np.roll(burn, (dr, dc), axis=(0, 1))
            new[(forest == TREE) & shifted] = BURNING
    return new

np.random.seed(1)
grid_size, density, steps = 21, 0.6, 10
forest = init_forest(grid_size, density)
frames = [forest.copy()]
for _ in range(steps):
    forest = step_fire_deterministic(forest)
    frames.append(forest.copy())

# Plot 4 representative frames
fig, axes = plt.subplots(1, 4, figsize=(14, 3.5))
for ax, idx in zip(axes, [0, 3, 6, 10]):
    ax.imshow(frames[idx], cmap=CMAP, vmin=0, vmax=3)
    ax.set_title(f'Step {idx}')
    ax.axis('off')
plt.suptitle('Deterministic Fire Spread')
plt.tight_layout(); plt.show()

Block 2：加入概率蔓延与随机起火

复杂化：现实中火苗不一定每次都能引燃相邻树木（受湿度、风力、距离影响），而且闪电等也会随机起火。我们引入：

  • p_spread：树被着火的邻居点燃的概率
  • p_ignite：每棵树每步自发燃烧的概率

动检：改变初始树密度，重复多次实验，画出“平均烧毁比例 vs 密度”曲线。你会看到类似逾渗（percolation）的相变：密度超过临界值后，烧毁比例急剧上升。

import numpy as np
import matplotlib.pyplot as plt
from matplotlib.colors import ListedColormap

EMPTY, TREE, BURNING, ASH = 0, 1, 2, 3
CMAP = ListedColormap(['white', 'green', 'red', 'black'])

def init_forest(grid_size, density, p_ignite):
    forest = np.zeros((grid_size, grid_size), dtype=int)
    forest[np.random.rand(grid_size, grid_size) < density] = TREE
    ignite = np.random.rand(grid_size, grid_size) < p_ignite * 10
    forest[(forest == TREE) & ignite] = BURNING
    return forest

def step_fire_prob(forest, p_spread, p_ignite):
    new = forest.copy()
    burn = (forest == BURNING)
    new[burn] = ASH
    # probabilistic spread from burning neighbors
    spread_risk = np.zeros_like(forest, dtype=float)
    for dr in (-1, 0, 1):
        for dc in (-1, 0, 1):
            if dr == 0 and dc == 0:
                continue
            spread_risk += np.roll(burn, (dr, dc), axis=(0, 1)).astype(float)
    catch = (np.random.rand(*forest.shape) < 1 - (1 - p_spread)**spread_risk)
    new[(forest == TREE) & catch] = BURNING
    # spontaneous ignition
    new[(forest == TREE) & (np.random.rand(*forest.shape) < p_ignite)] = BURNING
    return new

def run_fire(density, grid_size, p_spread, p_ignite, steps):
    forest = init_forest(grid_size, density, p_ignite)
    for _ in range(steps):
        forest = step_fire_prob(forest, p_spread, p_ignite)
    return (forest == ASH).sum() / forest.size

np.random.seed(2)
grid_size, steps, reps = 41, 30, 10
p_spread, p_ignite = 0.7, 0.001
densities = np.arange(0.1, 0.9, 0.03)
avg_burned = []
for d in densities:
    burned = [run_fire(d, grid_size, p_spread, p_ignite, steps) for _ in range(reps)]
    avg_burned.append(np.mean(burned))

fig, ax = plt.subplots(figsize=(8, 4))
ax.plot(densities, avg_burned, marker='o', color='firebrick')
ax.set_xlabel('Initial tree density')
ax.set_ylabel('Average burned fraction')
ax.set_title('Burned Fraction vs. Tree Density')
ax.grid(True, linestyle='--', alpha=0.5)
plt.tight_layout(); plt.show()

Block 3：完整动态森林——生长、火灾与长期演化

复杂化：树被烧掉后会变成空地，空地以一定概率重新长树；闪电随机点燃成熟森林。这样系统会进入“生长—火灾—再生”的循环，类似真实森林演替。

新增规则：

  • EMPTY 每步以概率 p_grow 长成 TREE
  • TREE 每步以概率 p_ignite 自燃
  • 火灾蔓延概率 p_spread 控制火势凶猛程度

分析：长时间运行后，森林面积会呈现剧烈振荡。可以画出 TREE / BURNING / ASH 三类细胞的数量时间序列，并观察不同 p_spread 下振幅与周期的变化。

import numpy as np
import matplotlib.pyplot as plt

from matplotlib.animation import FuncAnimation

EMPTY, TREE, BURNING, ASH = 0, 1, 2, 3

def init_full_forest(grid_size, density):
    forest = np.zeros((grid_size, grid_size), dtype=int)
    forest[np.random.rand(grid_size, grid_size) < density] = TREE
    return forest

def step_full_fire(forest, p_grow, p_spread, p_ignite):
    new = forest.copy()
    burn = (forest == BURNING)
    new[burn] = ASH
    # spread
    risk = np.zeros_like(forest, dtype=float)
    for dr in (-1, 0, 1):
        for dc in (-1, 0, 1):
            if dr == 0 and dc == 0:
                continue
            risk += np.roll(burn, (dr, dc), axis=(0, 1)).astype(float)
    catch = (np.random.rand(*forest.shape) < 1 - (1 - p_spread)**risk)
    new[(forest == TREE) & catch] = BURNING
    # spontaneous ignition
    new[(forest == TREE) & (np.random.rand(*forest.shape) < p_ignite)] = BURNING
    # regrowth
    new[(forest == EMPTY) & (np.random.rand(*forest.shape) < p_grow)] = TREE
    return new

6.2 Schelling 隔离模型（Schelling's Segregation Model）

Block 1：基础两类型隔离

场景：两种“居民”随机住在棋盘上。每个居民希望周围非空邻居中至少有比例 T 与自己同类；不满足时会搬到任意一个空房。即使 T 并不高（例如 30%），系统也会自发形成高度隔离的社区。

规则：

  • 遍历所有居民，判断是否“满意”：同类邻居 / 非空邻居 ≥ T
  • 不满意者随机搬到空房
  • 重复直到无人搬家或达到最大迭代次数

关键指标：平均相似度 = 所有居民同类邻居占全部邻居比例的平均。数值越高，隔离越明显。

import numpy as np
import matplotlib.pyplot as plt
import random

EMPTY = 0

def init_grid(grid_size, empty_ratio, type_ratios):
    total = grid_size * grid_size
    n_empty = int(total * empty_ratio)
    counts = [int(total * r) for r in type_ratios]
    # adjust rounding so total matches
    diff = total - n_empty - sum(counts)
    if diff > 0:
        n_empty += diff
    elif diff < 0:
        counts[-1] += diff
    cells = [EMPTY]*n_empty
    for t, n in enumerate(counts, start=1):
        cells += [t]*n
    random.shuffle(cells)
    return np.array(cells).reshape(grid_size, grid_size)

def avg_similarity(grid):
    size = grid.shape[0]
    total, n = 0.0, 0
    for r in range(size):
        for c in range(size):
            a = grid[r, c]
            if a == EMPTY:
                continue
            n += 1
            neighbors = []
            for dr in (-1,0,1):
                for dc in (-1,0,1):
                    if dr==0 and dc==0: continue
                    nr, nc = r+dr, c+dc
                    if 0 <= nr < size and 0 <= nc < size:
                        neighbors.append(grid[nr, nc])
            if neighbors:
                total += neighbors.count(a) / len(neighbors)
    return 100 * total / n if n else 0

def schelling_step(grid, tolerance):
    size = grid.shape[0]
    unhappy = [(r,c) for r in range(size) for c in range(size)
               if grid[r,c]!=EMPTY and not is_happy(grid,r,c,tolerance)]
    empty = [(r,c) for r in range(size) for c in range(size) if grid[r,c]==EMPTY]
    random.shuffle(unhappy); random.shuffle(empty)
    moves = 0
    for r,c in unhappy:
        if not empty: break
        nr, nc = empty.pop()
        empty.append((r,c))
        agent = grid[r,c]
        grid[nr,nc] = agent
        grid[r,c] = EMPTY
        moves += 1
    return grid, moves

def is_happy(grid,r,c,T):
    a = grid[r,c]
    if a == EMPTY: return True
    size = grid.shape[0]
    relevant = []
    for dr in (-1,0,1):
        for dc in (-1,0,1):
            if dr==0 and dc==0: continue
            nr,nc = r+dr, c+dc
            if 0 <= nr < size and 0 <= nc < size and grid[nr,nc] != EMPTY:
                relevant.append(grid[nr,nc])
    if not relevant: return True
    return relevant.count(a) / len(relevant) >= T

random.seed(4); np.random.seed(4)
grid_size, empty_ratio, T = 40, 0.2, 0.3
grid = init_grid(grid_size, empty_ratio, [0.4, 0.4])
print(f'Initial similarity: {avg_similarity(grid):.1f}%')

fig, axes = plt.subplots(1, 2, figsize=(12, 5))
axes[0].imshow(grid, cmap='viridis', vmin=0, vmax=2)
axes[0].set_title('Initial Random Mixing')
axes[0].axis('off')

for _ in range(50):
    grid, moves = schelling_step(grid, T)
    if moves == 0: break

axes[1].imshow(grid, cmap='viridis', vmin=0, vmax=2)
axes[1].set_title(f'Final Segregation (T={T})')
axes[1].axis('off')
print(f'Final similarity: {avg_similarity(grid):.1f}%')
plt.tight_layout(); plt.show()

场景：只有 A、B 两个站点。用户骑走一辆车后，可能还到 A，也可能还到 B。

import matplotlib.pyplot as plt

# Two-state transition matrix: rows = current station, cols = next station.
P = np.array([[0.7, 0.3],
              [0.4, 0.6]])
# Initial distribution: all bikes start at station A.
pi = np.array([1.0, 0.0])
steps = 30

# Iteratively apply the transition matrix.
history = [pi]
for _ in range(steps):

    history.append(pi)

eigvals, eigvecs = np.linalg.eig(P.T)
idx = np.argmin(np.abs(eigvals - 1))
stationary = eigvecs[:, idx].real
stationary = stationary / stationary.sum()

新增因素：

  • 站点容量上限

import numpy as np
import matplotlib.pyplot as plt

# Fix seed and initialize absolute bike counts and station capacity.
np.random.seed(42)
n_steps = 100
bikes = np.array([30, 10, 10])  # A, B, C initial counts
capacity = 40
P = np.array([
    [0.6, 0.3, 0.1],
    [0.2, 0.5, 0.3],
    [0.25, 0.25, 0.5]
])

history = [bikes.copy()]
empty_events = 0

for t in range(n_steps):
    # Each station may have one bike borrowed with probability 0.4.
    for i in range(3):
        if bikes[i] > 0 and np.random.rand() < 0.4:
            bikes[i] -= 1
            # Return the bike according to the transition matrix.
            dest = np.random.choice(3, p=P[i])
            if bikes[dest] < capacity:
                bikes[dest] += 1
            else:
                # If the destination is full, keep the bike at the origin.
                bikes[i] += 1
        elif bikes[i] == 0:
            empty_events += 1
    # Nightly rebalancing: redistribute evenly at midnight.
    if t % 24 == 0 and t > 0:
        avg = bikes.sum() // 3
        bikes = np.clip(np.array([avg, avg, avg]), 0, capacity)
    history.append(bikes.copy())

history = np.array(history)

# --- Visualizations ---
fig, axes = plt.subplots(1, 2, figsize=(13, 4))
for i, name in enumerate(['A', 'B', 'C']):
    axes[0].plot(history[:, i], label=name)
axes[0].axhline(capacity, color='red', linestyle='--', label='Capacity')
axes[0].set_title('Bike Counts with Rebalancing')
axes[0].legend()

    axes[0].plot(history[:, i], label=name, marker='o', markersize=3)
axes[0].set_title('Weather from Rainy Today')

Block 2：三状态晴/多云/雨模型

复杂化：把"多云"单独作为一个状态，让模型更精细。

状态空间：S = {Sunny, Cloudy, Rainy}

转移矩阵：3×3 矩阵，反映天气逐步过渡（雨 rarely 直接变晴）。

启发问题：状态越多，模型越准吗？会带来什么代价？

import numpy as np
import matplotlib.pyplot as plt

# Three-state transition matrix: Sunny, Cloudy, Rainy.
states = ['Sunny', 'Cloudy', 'Rainy']
P = np.array([
    [0.7, 0.2, 0.1],
    [0.3, 0.4, 0.3],
    [0.2, 0.3, 0.5]
])
# Today is cloudy.
pi = np.array([0, 1, 0])

# Evolve the distribution.
history = [pi]
for _ in range(50):
    history.append(history[-1] @ P)
history = np.array(history)

# Compute stationary distribution.
eigvals, eigvecs = np.linalg.eig(P.T)
idx = np.argmin(np.abs(eigvals - 1))
stationary = eigvecs[:, idx].real
stationary = stationary / stationary.sum()

# --- Visualizations ---
fig, axes = plt.subplots(1, 2, figsize=(13, 4))
for i, name in enumerate(states):
    axes[0].plot(history[:, i], label=name, marker='o', markersize=3)
axes[0].set_title('3-State Weather Evolution')
axes[0].legend()
axes[1].bar(states, stationary, color=['#fde047', '#93c5fd', '#60a5fa'], edgecolor='black')
axes[1].set_title('Long-Run Proportion')
plt.tight_layout(); plt.show()

Block 3：季节性非齐次 Markov Chain

复杂化：雨季和旱季的转移矩阵不同。例如雨季"雨天→雨天"概率更高，旱季"晴天→晴天"概率更高。

调整：

πt+1 = πt · Pseason(t)

启发问题：如果全年用一个矩阵，会低估还是高估雨天比例？如何确定季节切换点？

import numpy as np
import matplotlib.pyplot as plt

# Transition matrices for dry and wet seasons.
P_dry = np.array([
    [0.8, 0.15, 0.05],
    [0.4, 0.4, 0.2],
    [0.3, 0.4, 0.3]
])
P_wet = np.array([
    [0.4, 0.3, 0.3],
    [0.2, 0.4, 0.4],
    [0.2, 0.3, 0.5]
])
# Start on a sunny day.
pi = np.array([1, 0, 0])
history = [pi]

# Switch between dry and wet season matrices over one year.
for t in range(365):
    # Dry season roughly days 0-149 and 270-364; wet season in between.
    P = P_dry if (t % 365 < 150 or t % 365 >= 270) else P_wet
    pi = pi @ P
    history.append(pi)
history = np.array(history)

# --- Visualizations ---
fig, axes = plt.subplots(1, 2, figsize=(13, 4))
states = ['Sunny', 'Cloudy', 'Rainy']
for i, name in enumerate(states):
    axes[0].plot(history[:, i], label=name)
axes[0].set_title('Seasonal Non-Homogeneous Weather')
axes[0].set_xlabel('Day')
axes[0].legend()
axes[1].plot(history[:, 0], history[:, 2], color='purple', alpha=0.7)
axes[1].set_title('Sunny vs Rainy Phase Space')
axes[1].set_xlabel('Sunny'); axes[1].set_ylabel('Rainy')
plt.tight_layout(); plt.show()

Block 4：高阶 Markov Chain（考虑历史）

复杂化：真实天气有记忆——连续雨天后再下雨的概率更高。二阶 Markov 链把"今天+昨天"作为一个组合状态。

新状态空间：S' = {(今天, 昨天)}，共 9 个组合状态

转移：从 (昨天, 今天) → (今天, 明天)

启发问题：二阶模型一定比一阶好吗？什么情况下会过拟合？

import numpy as np
import matplotlib.pyplot as plt

# Three weather states: 0=Sunny, 1=Cloudy, 2=Rainy.
# Second-order transition matrix:
# rows = (yesterday, today) pair; cols = probability of tomorrow's weather.
# Pair index = yesterday * 3 + today.
P2 = np.array([
    # (S,S)  (S,C)  (S,R)  (C,S)  (C,C)  (C,R)  (R,S)  (R,C)  (R,R)
    [0.70,  0.40,  0.30,  0.50,  0.20,  0.20,  0.50,  0.30,  0.20],  # -> Sunny
    [0.20,  0.40,  0.40,  0.30,  0.60,  0.30,  0.30,  0.40,  0.30],  # -> Cloudy
    [0.10,  0.20,  0.30,  0.20,  0.20,  0.50,  0.20,  0.30,  0.50],  # -> Rainy
]).T  # transpose to 9 rows (pairs) x 3 columns (tomorrow)

# Verify every row sums to 1 (required for a valid transition matrix).
print('Row sums:', P2.sum(axis=1))

# Start: yesterday=0 (Sunny), today=1 (Cloudy) -> pair index 1.
state = 1
history = [1]

# Generate a 100-day sample path using the second-order transition rule.
for _ in range(100):
    # Sample tomorrow's weather from the distribution of the current pair.
    tomorrow = np.random.choice(3, p=P2[state])
    today = state % 3
    # New pair becomes (today, tomorrow).
    state = today * 3 + tomorrow
    history.append(tomorrow)

# --- Visualizations ---
fig, axes = plt.subplots(1, 2, figsize=(13, 4))
states = ['Sunny', 'Cloudy', 'Rainy']
axes[0].plot(history, marker='o', markersize=3)
axes[0].set_title('2nd-Order MC Sample Path')
axes[0].set_xlabel('Day')
axes[0].set_yticks([0, 1, 2])
axes[0].set_yticklabels(states)
unique, counts = np.unique(history, return_counts=True)
axes[1].bar(states, [counts[np.where(unique==i)[0][0]] if i in unique else 0 for i in range(3)],
            color=['#fde047', '#93c5fd', '#60a5fa'], edgecolor='black')
axes[1].set_title('Empirical Frequencies')
plt.tight_layout(); plt.show()

9. 三种随机建模视角对比

方法 核心思想 适用场景 输出
Monte Carlo 重复随机抽样，从大量试验中估计概率 风险分析、工期、落点、面积估计 结果分布、概率、置信区间
Agent-Based 个体简单规则 → 群体涌现行为 生态系统、人群、交通、商场 时序演化、相空间、涌现模式
Markov Chain 下一状态只依赖于当前状态 天气、共享单车、PageRank、NLP 状态概率演化、稳态分布
建模起手式：拿到带不确定性的题，先问： (1) 不确定变量是什么？适合什么分布？ (2) 个体之间有没有交互？（有 → ABS；没有 → Monte Carlo 或 Markov） (3) 未来状态是否只依赖当前？（是 → Markov） (4) 要估计的是概率/风险，还是长期稳定状态？
